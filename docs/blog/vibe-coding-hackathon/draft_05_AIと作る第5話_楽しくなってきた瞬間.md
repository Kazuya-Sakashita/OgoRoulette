# AIと作る ── 第5話「楽しくなってきた瞬間」

## 何かが変わったのはいつだったか

正確な日は分からない。

「作業が楽しい」と気づいたのは、ある夜の帰り道だった。
その日の朝、バグを直した。午後に機能を追加した。夜にデプロイした。

電車の中でスマホでアプリを開いて、友人にリンクを送った。
「試してみて」と。

その2分後に返信が来た。「これ本物のルーレットっぽい。」

思わず笑った。

---

## 「楽しい」がどこから来たのか

振り返ってみると、楽しくなったのに理由があった。

第3話の「すごい」の体験を繰り返すうちに、**Claude の使い方が分かってきた**。

最初は「コードを出してもらうツール」だと思っていた。
使い続けるうちに、それは違うと気づいた。

Claude は「一緒に考えるための対話相手」だった。

「この設計どう思う？」と投げると、問題点を指摘してくる。
「この方法と別の方法、どちらが良い？」と聞くと、比較してくれる。
「理解できないから説明して」と言うと、分解してくれる。

コードを出してもらうより、考えるプロセスに巻き込む使い方が、結果として良いものができた。

---

## 会話が変わっていた

最初のころの会話はこうだった。

```
自分：「ルーム機能を作ってください」
Claude：[コードを出す]
自分：「ありがとう」
```

数週間後の会話はこうなっていた。

```
自分：「参加者がリロードしたとき、Realtimeのチャンネルが切れることがある。
どういうケースが考えられる？」

Claude：「3つ考えられます。
1. WebSocketがタイムアウト
2. ネットワーク切断後の再接続
3. ページがバックグラウンドに移動したとき」

自分：「ケース3が一番厄介そう。どう対処する？」

Claude：「スマホでは……」
```

質問の仕方が変わっていた。
「作って」ではなく「考えて」になっていた。

---

## 「作れなかったものが作れた」

ある日、Lighthouse を開いた。

```
Performance: 91
Accessibility: 92
Best Practices: 100
SEO: 91
```

最初に計測したときは 62 だった。

技術的なことだけではない。デザインも変わった。
スマホで見たときに「これ普通のアプリじゃないみたい」と言われるようになった。

Framer Motion でルーレットのアニメーションを作った日、スマホで動かして「きれい」と思った。
自分が書いたアニメーションではなかった。
でも自分が「こういうふうに動いてほしい」と伝えたものだった。

「自分が作れなかったものが、自分のアイデアのまま形になった」

これが楽しさの正体だと思った。

---

## 友人たちが使う日

リリースして最初の「本番」は、会社の飲み会だった。

「今日の誰が奢るか、これで決めよう」と言ってアプリを開いた。

「なにこれ、ちゃんとしてる。」
「操作できないようになってるの？」
「あ、ルーレット回ってる。全員に見えてる。」

5人でひとつの画面を囲んで、ルーレットが止まる瞬間を待った。
止まった。

「うわー！」

そのリアクションは、じゃんけんでも指差しでも出てこないものだった。

---

## Claude は覚えていない

その夜、家に帰ってターミナルを開いた。

新しいセッション。Claude には「今日うまくいった」は分からない。
「うわー！」のリアクションも知らない。
友人が5人で囲んで見ていたことも。

でも、それでいいと思えるようになっていた。

Claude はツールでも仲間でもなく、**その瞬間のセッションで最善を尽くすパートナー**だ。
覚えていないのは仕様で、それを補うのが CLAUDE.md であり、自分の文脈管理だ。

チームで仕事をするとき、全員が全部を共有している必要はない。
自分がコンテキストを持って、うまく引き継ぐことができれば良い。

---

## 楽しくなってきた瞬間の正体

「楽しい」には2種類あった。

ひとつは「速く作れる楽しさ」。これは最初から少しあった。
もうひとつは「自分だけでは辿り着けなかった場所に辿り着く楽しさ」。

後者は、使い始めて数週間経ってから来た。

自分の考えを言語化して、対話の中で精度を上げて、コードに落として、動かして、直す。
そのサイクルを回すこと自体が、面白くなってきた。

作るものが「アプリ」ではなく「自分の考え」に近くなってきた感覚があった。

---

## このシリーズを書いた理由

「AIとアプリを作る」という体験を記録したくなった理由はひとつだ。

**「革命だ」でも「大したことない」でもなかった**、と伝えたかった。

違和感があった。壊れた日があった。「自分のコードではない」と感じた日があった。

それを乗り越えた先に、楽しさがあった。

Claude Code は、使い始めればすぐに魔法のように動くわけではない。
一緒に仕事をする相手として扱うための、自分なりの方法を見つける必要がある。

その過程が、開発という体験そのものだった。
