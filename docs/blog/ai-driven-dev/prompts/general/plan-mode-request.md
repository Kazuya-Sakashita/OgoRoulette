# 実装前に設計を確認させる（PlanModeリクエスト）

**レベル:** 中級  
**再利用性:** 高  
**効果:** ★★★★☆  
**タグ:** `テンプレ化`

---

## 目的

非自明な実装変更をする前に、AIに設計を提示させて人間が承認する。  
「実装してから設計ミスに気づく」ことを防ぐ。

## 使用タイミング

- 複数ファイルに跨がる変更をするとき
- 実装方針が複数考えられるとき
- リスクのある変更（本番影響あり）をするとき

---

## プロンプト

```
以下の変更を実施する前に、実装計画を立案してください（EnterPlanMode）。

目的: {変更の目的}

変更スコープ:
- 変更ファイル: {ファイル一覧}
- 変更内容: {概要}
- 変更しないもの: {対象外の明示}

制約:
- {制約1（例: モバイルへの影響なし）}
- {制約2（例: 新規依存追加なし）}
- {制約3（例: 変更ファイルは1つ）}

計画には以下を含めてください:
1. 具体的な変更箇所（ファイル名・行数）
2. 変更前後のコードイメージ
3. リスクと対策
4. 検証方法

計画を見てから実装の承認を判断します。
```

---

## 工夫ポイント

- 「変更しないもの」を明示することでスコープが勝手に広がらない
- 「計画を見てから承認」と明示することで、AIが実装を急がない
- 検証方法まで計画に含めることで、実装後の確認を忘れない

## 改善余地

- 計画の承認後に「適度な粒度でコミットしてください」を追加で指示するとコミット履歴が綺麗になる

## 実行結果

PCの2カラムレイアウト変更時に使用。  
計画段階で「左カラムをstickyにするか」「dividerをどこに入れるか」が決まり、  
実装後の手戻りがゼロだった。
