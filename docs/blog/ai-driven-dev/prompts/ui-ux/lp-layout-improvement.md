# LP のUX課題を評価して改善提案を出させる

**レベル:** 中級  
**再利用性:** 高  
**効果:** ★★★★☆  
**タグ:** `テンプレ化`

---

## 目的

LPのUXを定量・定性の両面で評価し、  
優先度付きの改善案リストを生成する。

## 使用タイミング

- LPのコンバージョンが低いとき
- 「なんか微妙」だけどどこを直すか分からないとき
- A/Bテストの候補を洗い出したいとき

---

## プロンプト（UX評価）

```
以下のLPを評価してください。

URL: {lp-url}

評価軸:
1. ファーストビューでのメッセージ明確さ（10点満点）
2. CTAの配置と視認性（10点満点）
3. 社会的証明の有無と説得力（10点満点）
4. モバイルでの操作性（10点満点）
5. 読み込み速度の印象（10点満点）

各軸のスコアと改善提案を出力してください。
改善提案は「変更ファイル名・変更箇所・変更内容」まで具体化してください。
```

## プロンプト（CTAの順序改善）

```
{file-path} のCTA配置を改善してください。

現状の問題:
- バイラルCTA「友達に教える」がゲスト→ログインCTAの下にある
- ゲストユーザーがバイラルCTAに気づかず離脱している

改善方針:
- バイラルCTAを全ユーザー向けに上に移動
- ゲスト向けログインCTAは下に配置（ゲストのみ表示）
- ログインCTAのコピーを「機能訴求」から「損失回避」に変更

変更前コピー例:「次回もすぐに使える」
変更後コピー例:「この履歴、端末が変わると消えます」

理由: スピン直後の「今この瞬間のデータが消えるかも」という危機感は
      抽象的な機能説明より転換率が高い。

実装: {file-path} のみを変更してください。
```

---

## 工夫ポイント

- 「損失回避」コピーの根拠を説明することで、AIが適切な言い換えをしてくれる
- 評価軸を10点満点で指定することで、後から比較ができる
- 「変更ファイル名まで具体化」と要求することで実装できる粒度の提案が返ってくる

## 改善余地

- 実際のコンバージョンデータがある場合は渡すとより精度が上がる
- gstack browse と組み合わせてスクリーンショット付きで評価させると説得力が増す

## 実行結果

- WinnerCard CTA順序変更（ISSUE-145）: 5分で実装完了
- バイラルCTAの位置が改善され、ゲストユーザーに確実に届くようになった
