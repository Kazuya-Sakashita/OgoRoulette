# モバイルファーストアプリをPCで2カラム化する

**レベル:** 中級  
**再利用性:** 高  
**効果:** ★★★★☆  
**タグ:** `テンプレ化` `おすすめ`

---

## 目的

モバイル向けに設計されたシングルカラムLPを、  
モバイルに一切影響を与えずにデスクトップで2カラムレイアウトに変更する。

## 使用タイミング

- モバイルファーストで作ったLPをPCでも良く見せたいとき
- `max-w-md` のコンテンツがPC画面の中央で浮いている状態を改善したいとき

---

## プロンプト

```
{file-path} のレイアウトを以下の条件でPC対応してください。

現状の問題:
- max-w-[360px] md:max-w-md のシングルカラム
- 1440px幅では448pxコンテンツが中央に浮いて「未完成なモバイルアプリ」に見える

要件:
1. モバイル（< 1024px）: 変更なし
2. デスクトップ（≥ 1024px）: 2カラムレイアウト

左カラム（sticky）:
  - {左カラムに入れるコンポーネント一覧}
  - 幅: lg:w-[480px]
  - スクロール時固定: lg:sticky lg:top-24

右カラム:
  - {右カラムに入れるコンポーネント一覧}
  - 幅: lg:flex-1

実装方法:
- Tailwindの lg: プレフィックスのみ使用
- 新規コンポーネント作成禁止
- 変更ファイル: {file-path} のみ

コンテナ変更:
- max-w-md → max-w-md lg:max-w-5xl
- flex-col → flex-col lg:flex-row lg:items-start lg:gap-16

実装前に設計を提示して承認を得てください（EnterPlanMode）。
```

---

## 工夫ポイント

- `lg:` プレフィックスのみ使用と明示することでモバイル崩壊を防ぐ
- 「変更ファイル1つのみ」と制約を付けることでスコープが広がらない
- `EnterPlanMode` を明示することで、実装前に設計確認ができる
- 左カラムを `sticky` にする設計を最初から指定しておくと後から変えにくい

## 改善余地

- `isDesktop` の検出方法（`useMediaQuery` vs `useRef` + `resize`）をプロンプトで指定しないと、SSRで問題が出る実装をAIが選ぶことがある
- 仕切り線（`hidden lg:block w-px bg-white/10`）を要件に含めると完成度が上がる

## 実行結果

- 変更ファイル: `app/page.tsx` のみ
- 追加行数: 約30行（Tailwindクラス + div追加）
- モバイルへの影響: ゼロ（実機確認済み）
- 1440px での表示: 左カラムsticky + 右カラムフレキシブル
