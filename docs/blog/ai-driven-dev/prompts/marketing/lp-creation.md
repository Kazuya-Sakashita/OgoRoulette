# アプリのLPをHTMLで一から生成する

**レベル:** 上級  
**再利用性:** 中  
**効果:** ★★★★☆

---

## 目的

アプリの特徴・ターゲット・競合情報を渡し、  
スタンドアロンHTMLのLPを生成する。

## 使用タイミング

- アプリのマーケティングページが必要なとき
- 外部公開用のLPをアプリ本体と分離して作りたいとき
- PDFやスクリーンショットとして配布したいとき

---

## プロンプト

```
以下の情報をもとに、アプリのLPをスタンドアロンHTMLで作成してください。

【アプリ情報】
アプリ名: {アプリ名}
URL: {アプリURL}
一行説明: {アプリの説明}
主な機能: 
  - {機能1}
  - {機能2}
  - {機能3}

【ターゲット】
{例: 飲み会・合コン・社内ランチで使えるグループ向けアプリ}

【LPの構成】
1. ヒーロー（キャッチコピー + CTA）
2. 主要機能の説明（3〜4項目）
3. 使い方（ステップ形式）
4. 社会的証明（利用シーン・数字）
5. CTA（再訴求）

【デザイン要件】
- カラーパレット: {メインカラー}
- スタイル: {例: モダン・ダーク・モバイルファースト}
- フォント: Google Fonts（{フォント名}）
- スタンドアロンHTML（外部CSS依存なし）

【出力先】
docs/lp/index.html
```

## プロンプト（PDF出力用調整）

```
docs/lp/index.html をPDFとして出力するスクリプトを作成してください。

要件:
- Playwrightを使用
- 単一の長いページとして出力（ページ区切りなし）
- モバイル幅（480px）でレンダリングしてからA4幅（794px）にスケールする
- フォント・背景色が正確に出力される（print-color-adjust: exact）
- アニメーションをすべて停止してから出力する

出力先: docs/lp/ogoroulette-lp.pdf
スクリプト: docs/lp/generate-pdf.mjs
```

---

## 工夫ポイント

- 「スタンドアロンHTML」と指定することで外部CDN依存がなくなり、PDFやオフライン配布に使える
- PDF出力は「モバイル幅でレンダリング→A4幅にスケール」の方法が最も品質が高い（ページ区切りなし）
- `print-color-adjust: exact` を明示しないとブラウザが背景色を省略する

## 改善余地

- アニメーションを動かしたままスクリーンショットを撮りたい場合は別途スクリプトが必要
- OGP画像としてキャプチャする場合は `--clip` オプションで切り抜く

## 実行結果

- `docs/lp/index.html` として生成、Vercelで `public/lp.html` として公開
- PDFは466px幅 → 794px幅（1.65倍スケール）で単一ページとして出力
- ページ区切り問題を解決するまでに2回の試行が必要だった
