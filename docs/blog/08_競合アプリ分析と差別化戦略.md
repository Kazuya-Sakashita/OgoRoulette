# 競合アプリ分析と差別化戦略

## 類似アプリを調べた

「誰が奢るか決める」系のアプリ・機能は複数存在する。

代表的なもの：

| アプリ / 機能 | 形式 | 特徴 |
|-------------|------|------|
| Googleルーレット | Web検索 | 「ルーレット」と検索すると出る。名前を入力して回す。 |
| Wheel of Names | Web | 名前をカンマ区切りで入力。シンプルで機能豊富。 |
| LINE グループ機能 | SNS内 | グループ内でランダム選択できる機能がある。 |
| random.org | Web | 本格的なランダム選択。視覚的な演出はほぼない。 |
| 奢り決定系アプリ（複数） | スマホアプリ | ランダム選択のみ。金額計算機能はまちまち。 |

## 競合の強みと弱み

### Googleルーレット

**強み**：
- 検索するだけで使える。最高の導線。
- 認知度が圧倒的に高い。

**弱み**：
- 「誰かの画面で回す」→ 結果が一人にしか見えない
- 金額計算はない
- 「みんなで盛り上がる」体験設計ではない

### Wheel of Names

**強み**：
- カスタマイズ性が高い（色・サウンド・セグメント数）
- シェア機能がある

**弱み**：
- URLを共有しても全員の画面で同時演出はない
- 「飲み会で使う」には少し重い UI

### LINE グループ

**強み**：
- すでにグループメンバーが全員いる
- 通知が飛ぶ

**弱み**：
- 演出がほぼない
- 金額計算がない
- LINEグループに縛られる

## 差別化ポイント

OgoRoulette が目指す独自価値は1つ：

> **「全員の画面で同時に盛り上がる」体験**

既存アプリは「誰か1人の画面でやる→見せる」モデル。
OgoRoulette は「QRコードで全員が同じ画面に入る→全員の前で回る」モデル。

この差は大きい。

### なぜ「全員参加」が重要か

- 「誰かの画面を見せてもらう」は他人事感がある
- 自分のスマホで同じ瞬間を見ると当事者感が増す
- 「見てた」証拠がある→「不正じゃない」信頼感

この体験は Wheel of Names や Google ルーレットでは実現できない。

---

## 「一部奢り」機能の差別化

「全額奢り」ではなく「一部だけ奢る」ケースが現実には多い。

例：
- 「3万円の中から1万円だけ出す」
- 「飲み放題代（5000円）だけ奢る、食事代は割り勘」

既存アプリはこの計算を一切サポートしていない。

OgoRoulette の金額計算：

```
合計: 30,000円
奢り: 10,000円
当選者が払う: 10,000円
残り: 20,000円 ÷ (参加者-1)人 = 端数切り上げ
```

この計算を自動でやって全員に表示する。
「で、私いくら払えばいいの？」問題を解決する。

## OgoRoulette が負けている点（正直に）

### 1. 認知度がゼロ

Googleルーレットは検索するだけで出てくる。
OgoRoulette を知ってもらうには「飲み会でその場で使ってもらう」しかない。
つまり口コミ・バイラル設計が必須。

**課題**：
- 結果のシェア機能（X, LINE）はあるが、使われているか不明
- 「友達に教えたくなる」仕掛けが弱い

### 2. ゲスト体験の複雑さ

Googleルーレットは「検索して使う」だけ。
OgoRoulette は「QRコードでアクセスしてルームに入る」というステップがある。

一部のユーザーには「めんどくさい」に見える可能性。

**課題**：
- Join フローをどれだけシンプルにできるか
- ニックネーム入力すら不要にできないか

### 3. ルーム作成のコスト

飲み会の幹事がその場でスマホを操作してルームを作る必要がある。
「事前準備なし」で使えるが、飲み会が始まった後に「ちょっと待って、セットアップするから」という瞬間が発生する。

**課題**：
- ルーム作成 → 招待 → 全員参加 の最短ステップを計測する
- 現状何ステップか、理想は何ステップか

## 差別化戦略まとめ

「ルーレットツール」ではなく「飲み会のエンタメ」として位置づける。

競合が「便利さ」を売るなら、OgoRouletteは「盛り上がり」を売る。

### 守るべき差別化要素

1. **リアルタイム同期**（全員の画面で同じ瞬間を体験）
2. **金額計算の自動化**（「いくら払えばいいか」を即答）
3. **演出の品質**（confetti、WinnerCard の段階的演出）

### 改善が必要な差別化要素

1. **招待の手軽さ**（QRコードは今でも十分だが、さらに短縮できないか）
2. **シェアのしやすさ**（結果を友達に送ると次の参加者が増える）
3. **再利用のしやすさ**（「次回の飲み会でもこのアプリ使おう」）

---

## 学び

> **差別化は「競合より機能が多い」ではない。「この体験は競合では得られない」という1点を作ることだ。**

> **OgoRoulette の1点は「全員の画面で同時に盛り上がる」。この体験を壊さない設計判断を常にする。**
