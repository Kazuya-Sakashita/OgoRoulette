アプリが動くようになった後、「伝える」フェーズで使うプロンプト群。

## 記事タイプ
体験記 — 開発・学習・プロジェクトを通じた経験談

## 核心メッセージ（1文で）
Supabase Realtimeへの移行は3秒ポーリングよりも体験が良くなるが、「missed events（再送なし）」という落とし穴があり、ポーリングフォールバックを最初から設計に組み込む必要があった。

## 実際に起きた出来事（体験の素材）
- ルーム内でオーナーがSPINしたとき、一部のメンバー画面でアニメーションがスキップされた
- 調査したところ、Supabase Realtime の WebSocket が切断した際に missed events が再送されない仕様だった
- 10秒ポーリングをフォールバックにしていたが、SPIN_COUNTDOWN_MS（3秒）より遅くなるケースで elapsed >= TOTAL_ANIM_MS になりアニメーションをスキップする設計になっていた
- spinStartedAt を元に elapsed を計算し、フォールバック時でもアニメーションを再生できる設計に修正した

## コード・数値
- 修正前: ポーリング fallback で 100% アニメーションスキップが発生
- 修正後: elapsed が TOTAL_ANIM_MS 未満であればアニメーションを再生
```typescript
// 修正前（fallback で必ずスキップになっていた）
if (elapsed >= TOTAL_ANIM_MS) {
  setPhase("result")  // スキップ
  return
}

// 修正後（残り時間に合わせて再生）
const remainingMs = TOTAL_ANIM_MS - elapsed
setSpinElapsedMs(elapsed)
setPhase("spinning")
setTimeout(() => setPhase("result"), remainingMs)
```

## 対象読者
Next.js + Supabase を使って個人開発をしている人。Realtime は使い始めたが落とし穴にはまった経験がある人。

- 余計な前置きなしで本文から始める
