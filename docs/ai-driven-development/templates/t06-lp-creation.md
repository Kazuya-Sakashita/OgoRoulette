# ランディングページ生成テンプレート (t06)

## 使うタイミング
アプリが動くようになったあと、外部に公開するためのLP（ランディングページ）を作るとき。AIに1ファイルの完結したHTMLとして生成させる。

## 重要度
`★★★☆☆ MEDIUM` — 機能開発よりも後になるが、ユーザー獲得・信頼形成に直結する。品質はLighthouse 95+を目標にする。

## テンプレート

```
以下の情報をもとに、Webアプリのランディングページ（LP）を1つのHTMLファイルとして生成してください。

---

## アプリ情報

### アプリ名
[例: OgoRoulette]

### アプリURL（CTAリンク先）
[例: https://ogo-roulette.vercel.app/]

### ターゲットユーザー
[例: 飲み会・ミーティング・ゲームグループなど、グループで物事を決める機会がある人]

### Before（課題）
[ユーザーが今感じている痛み。例: じゃんけんは不公平、声の大きい人が決めてしまう、毎回もめる]

### After（価値）
[アプリを使った後の理想状態。例: 誰でも納得できる公平な抽選が5秒でできる]

### 主要機能（3つ）
1. [例: 名前を入力してルーレットを回すだけ — シンプル操作]
2. [例: スマホでもPCでも使える — デバイス不問]
3. [例: インストール不要・会員登録不要 — 今すぐ使える]

### CTA（行動喚起）テキスト
[例: 今すぐ無料で使う]

### カラーテーマ
- メイン: [例: #3B82F6（青）]
- アクセント: [例: #F59E0B（黄）]
- 背景: [例: #FFFFFF]

### トーン・スタイル
[例: カジュアル・明るい・日本語メイン・余白多め]

---

## 技術要件

- 単一HTMLファイル（外部CSSフレームワーク・外部JSライブラリのCDN読み込み禁止）
- スタイルは `<style>` タグ内にインライン記述
- インタラクションは最小限のVanilla JSのみ
- モバイルファースト（375px以上で崩れないこと）
- 画像はSVGまたはCSSのみで表現（外部画像URL禁止）
- Lighthouse スコア目標: Performance 95+ / Accessibility 95+ / SEO 90+

## ページ構成（セクション順）
1. ヒーロー — キャッチコピー + サブコピー + CTAボタン
2. 課題提起 — Before の痛みを共感的に描写
3. 解決策 — After の理想とアプリの位置づけ
4. 主要機能 — 3つのFeatureカード
5. 使い方 — 3ステップで使える流れ（ステップ図）
6. CTA — 最後の行動喚起
7. フッター — コピーライト

---

生成後に以下を確認してください:
- [ ] `<html lang="ja">` が設定されているか
- [ ] 全画像・SVGに `alt` 属性があるか
- [ ] CTA ボタンに `aria-label` があるか
- [ ] `<meta name="viewport">` に `maximum-scale` がないか（A11y違反）
- [ ] カラーコントラスト比 4.5:1 以上を満たしているか
```

## 使用例（OgoRouletteでの実績）

`public/lp.html` はこのテンプレートで生成した。「外部CDN不使用・単一HTML・Lighthouse 95+」の制約を明示したことで、FontAwesomeやBootstrapへの依存なく軽量なLPが生成された。後日 ISSUE-147 で `maximum-scale=1` を削除した経験から、テンプレートにA11yチェックリストを追加した。

## ポイント
- **外部CDN不使用**の制約は必ず入れる。入れないとAIがBootstrap等を使い、パフォーマンスが落ちる
- カラーテーマを数値で指定するとコントラスト計算がしやすい
- 「Before/After」の構造はコピーライティングの基本。これを埋めるだけでAIが説得力のある文章を書く
- 生成後は `docs/lp/generate-pdf.mjs` のようなスクリプトでPDF化しておくと、非エンジニアへの共有が楽
- LPは機能追加のたびに更新する。古い情報が掲載されたままにしない
