# gstack品質レビューテンプレート (t05)

## 使うタイミング
機能実装が一段落したタイミング、リリース前、または「なんとなく気になる品質問題がある」と感じたとき。AIにヘッドレスブラウザでアプリを検査させ、網羅的な品質ISSUEリストを得る。

## 重要度
`★★★★★ CRITICAL` — 人間の目視レビューでは見落とすパフォーマンス・アクセシビリティ・SEOの問題をAIが自動検出する。リリース前に必ず実施する。

## テンプレート


## 使用例（OgoRouletteでの実績）

リリース前の品質チェックで使用。ISSUE-147〜150はこのテンプレートのレビュー結果から抽出された。具体的には「viewport の maximum-scale=1 がユーザーのピンチズームを妨げている（A11y P0）」「Noto Sans JP の全ウェイト読み込みによるLCP悪化（パフォーマンス P1）」「muted テキストのコントラスト比不足（A11y P1）」「RouletteWheel の動的 import 未実施（パフォーマンス P1）」が検出された。

## ポイント
- 評価観点のチェックボックスは、時間がなければP0に直結する「パフォーマンス」「A11y」「エラーハンドリング」の3つだけでも実施する価値がある
- ISSUEテーブルは `docs/issues/` 配下にISSUE番号付きファイルとして保存し、GitHubのISSUEと紐づけると追跡しやすい
- P0（クリティカル）の問題は次のコミットまでに必ず修正する
- 「既知の問題」を明示することで、AIが同じことを何度もISSUEとして挙げるのを防ぐ
- モバイル幅（375px）での検査は必須。デスクトップのみのレビューでは実際のユーザー体験を正確に評価できない
