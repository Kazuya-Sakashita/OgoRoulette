# UIコンポーネント生成テンプレート (t02)

## 使うタイミング
新しいUIコンポーネントをゼロから作るとき。ページ全体ではなく**コンポーネント単位**でAIに依頼する際に使う。「なんとなく作って」は禁止。このテンプレートで仕様を明文化してから依頼する。

## 重要度
`★★★★☆ HIGH` — 曖昧な依頼はAIが勝手に判断して既存コードを書き換えるリスクがある。コンポーネント境界を明示することで被害を最小化できる。

## 使用例（OgoRouletteでの実績）

`RouletteWheel` コンポーネントの実装時に使用。「ルーレットを描画するコンポーネント」だけでは曖昧だったため、「Canvas APIを使う」「スピン速度は線形に減速する」「外部アニメーションライブラリ禁止」といったデザイン要件を明記することで、意図通りの実装が一発で得られた。

## ポイント
- **ページ単位ではなくコンポーネント単位**で依頼する。大きい粒度ほどAIが独断で決める範囲が増える
- Props定義は事前に書く。「AIに決めてもらう」と型が曖昧になり後でリファクタが必要になる
- 「参考にしてほしい既存コード」は必ずパスで示す。AIがプロジェクトのスタイルに合わせてくれる
- 制約の2行（依頼箇所以外変更禁止・実装前確認）は削除しないこと。これがないとAIが勝手にリファクタを始める
- 生成後は必ず `Props定義` と `アクセシビリティ（aria属性）` を人間がレビューする
