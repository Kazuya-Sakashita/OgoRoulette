## クイックスタート

### 今日から使い始める

1. **CLAUDE.md を書く**（セッションをまたいでも記憶を維持する）

```markdown
# プロジェクト名 — 開発ルール

## 絶対に守ること
- 頼んだ箇所以外は変更しない
- 実装前に「何を・なぜ・どう実装するか」を一言確認する
- 理解できないコードは書かない

## 禁止事項
- router.back() 禁止（Link href を使う）
- [過去に問題になったパターン]
```

2. **T-01（要件定義テンプレ）でMVPを絞る** → [`templates/t01-requirements.md`](templates/t01-requirements.md)

3. **最強ループを1周回す**
   - gstack評価（T-05）→ 改善案 → 実装 → 再評価

---

## 最強ループ

```
① gstack（全体評価）
      ↓
② HEART（UX分解）
      ↓
③ 改善案生成（AI）
      ↓
④ 実装（AI + テンプレート）
      ↓
⑤ 再評価
      ↓
  ① に戻る（品質が積み上がる）
```

**ポイント：1ループ = 1〜2個の改善に絞ること**

## 特に重要なテンプレート TOP 5

| # | テンプレート | なぜ重要か |
|---|---|---|
| T-01 | 要件定義 | 最初に使う。MVPを絞らないとすべてが崩れる |
| T-03 | デバッグ | ログ+コードを渡すだけで調査時間が1/3になる |
| T-05 | gstackレビュー | 品質を数値化して改善ループを回せる |
| T-02 | UI生成 | 禁止事項を入れることで意図しない変更を防ぐ |
| T-07 | ブログ生成 | 体験を記事化してコミュニティと繋がれる |

## AI駆動開発の本質

> AIは「速く書けるツール」ではなく、
> **「自分が言語化できていないことを実装に落としてくれるパートナー」**だ。
>
> 何を作るか・なぜ作るか・品質を判断するのは常に人間の仕事。
