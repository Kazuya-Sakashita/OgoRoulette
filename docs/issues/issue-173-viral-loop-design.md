# ウイルスループ設計（シェアから新規ユーザー獲得へ）

## 概要

OgoRoulette はシェアされやすい体験を持っているが、シェアした先のユーザーが「自分のグループでも使おう」とならない導線が不足している。シェア → 閲覧 → 利用 → 再シェアという自己増殖ループを設計する。

## 背景

現在の Growth スコアが 48 点と最も低い主因は「ウイルスループがない」こと。

- ルーレット結果はシェアされる機会がある（Phase A 1.5s でシェアボタン）
- しかし現在のシェア内容は「OgoRoulette で奢りを決めました + URL」という静的テキスト
- URL を見た人が「なにこれ？」と感じてアクセスしても、ウェルカムページで止まる
- 「自分のグループで使おう」というアクションに繋がる動線が弱い

## 現状の問題

1. シェアテキストに当選者名・状況が入っていない → 受け取った人に内容が伝わらない
2. シェアされた URL からアクセスしたユーザーへの「今すぐ作る」導線が弱い
3. 参加者（メンバー）が自分でホストになる導線がない（使ってみて気に入ったのに、次回自分が使う方法を知らない）
4. グループの中で「誰かがOgoRouletteを使ったことがある」状態から「自分も使う」への転換が設計されていない

## 目的

- シェアされたコンテンツが「次のユーザー獲得」に繋がる仕組みを作る
- 参加者（ゲスト）が次回ホストになれるような誘導を作る
- 「見た人が使いたくなる」シェアコンテンツを設計する

## 対応内容

### 1. シェアテキストの動的化

```
現在: 「OgoRoulette で奢りが決まりました！ https://ogo-roulette.vercel.app」

改善:
「${winnerName}さんが${memberCount}人分の奢りを担当することになりました 🎰
OgoRoulette で公平に決めました（全員の画面で同時に動くやつ）
↓ 次のランチで使ってみる
https://ogo-roulette.vercel.app」
```

「全員の画面で同時に動く」という体験の価値をテキストに含める。

### 2. メンバー → ホスト転換の導線

WinnerCard Phase B に参加者（メンバー）向けのセクションを追加:
```
「次は自分でルームを作ってみる？→ グループを作る」
```
このボタンでゲストでもルーム作成ページへ誘導（ゲスト作成は既に可能）。

### 3. シェアされた URL からの流入最適化

ウェルカムページに「ルーレットで盛り上がりたい？」という**体験起点のコピー**を追加。
現在の「保存・共有・履歴が使えます」よりも「まず試してみる」を前面に。
デモルーレットを回すと「グループでやってみる」CTAが出る（既存だが強化）。

### 4. 招待リンクの活用（中長期）

ルームリンク（`/room/[code]`）を共有することで、アクセスした人が自動的に参加者になれる。
「このリンクを送れば誰でも参加できます」というメッセージを作成直後に表示。
→ 既存機能（QR + ルームコード）を「招待リンク」として明示的に打ち出す。

## 完了条件

- [ ] シェアテキストに当選者名・参加人数・体験の説明が含まれる
- [ ] WinnerCard Phase B のメンバービューに「ルームを作る」リンクがある
- [ ] ウェルカムページの「まず試してみる」CTAが最上位アクションになっている
- [ ] ルーム作成直後に「このリンクを送ると全員が参加できます」の説明がある

## 優先度

**Nice-to-have** — 構造的に大きな改善だが、Phase 1・2 の信頼性と体験改善を先に行う。

## 期待効果

- Growth スコア: 56（Phase 1 後）→ 64（+8）
- Product スコア: 71 → 74（+3）
- 自然増加ユーザー数の向上
