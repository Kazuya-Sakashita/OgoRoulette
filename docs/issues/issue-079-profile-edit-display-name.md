# ISSUE-079: プロフィール編集導線の追加（display_name 変更）

## ステータス
✅ 完了（feat: ISSUE-078〜079 + fix: ISSUE-106）

## 優先度
Medium

## カテゴリ
Frontend / UX

## 概要
ログインユーザーが `display_name`（公開名）をいつでも変更できる導線を追加する。

## 実装方針

### 導線
`/home` ヘッダーのユーザーアイコン / ログアウトボタン横に「プロフィール」リンクを追加。
または `/home` 内のボトムシートとして実装する（新規ページ作成を避けるため）。

### UI（最小実装）

```
┌────────────────────────────────┐
│ ⚙ プロフィール                  │
│                                │
│ 公開名（SNSに表示される名前）    │
│  ┌──────────────────────────┐  │
│  │  ユーザー3a2f             │  │
│  └──────────────────────────┘  │
│  ※ 本名ではなくニックネームを    │
│    設定することをお勧めします    │
│                                │
│  [保存する]                     │
└────────────────────────────────┘
```

### API

```
PATCH /api/profile
Body: { displayName: string }
```

## 受け入れ条件
- [ ] ログインユーザーが display_name を変更できる
- [ ] 変更後は即座に全公開箇所に反映される
- [ ] 空文字保存時は fallback に戻る
- [ ] provider_name（本名）は表示・変更不可
