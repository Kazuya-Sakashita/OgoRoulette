# ISSUE-224: UX改善(P1) — エラーメッセージを具体的にし、リカバリー導線を追加する

## ステータス
📋 未着手

## 優先度
**P1 / High** — Task Success スコアとG-STACK信頼性の主要ボトルネック

## カテゴリ
UX / Error Handling / Clarity / Reliability

## 対象スコア
HEART-Task success: +1 / G-STACK-Clarity: +1 / G-STACK信頼性: +1 → 総合 +1〜2点

---

## Summary

アプリ全体でエラーメッセージが「○○に失敗しました」という汎用文言で終わり、
ユーザーが何をすればよいか分からない。
飲み会の場で突然「ルームの読み込みに失敗しました」と表示されても、
同席者の前で操作方法を調べる余裕はない。

---

## Background

現在のエラー表示箇所と問題:

| 箇所 | 現在のメッセージ | 問題 |
|------|----------------|------|
| ルーム作成失敗 | "ルームの作成に失敗しました" | なぜ失敗したか不明。再試行ボタンなし |
| スピン失敗 | "スピンに失敗しました" or "サーバーエラー" | リカバリー手順なし |
| ルーム読み込み失敗 | "ルームの読み込みに失敗しました。ルーレットの結果は保存されています。再試行してください。" | "結果は保存"は嘘の可能性あり（スピン失敗時は未保存）|
| ネットワークエラー | "ネットワークエラーが発生しました" | ユーザーは何をすればよいか分からない |
| ChunkLoadError | グローバルエラー境界発火 → 自動リロード | リロード回数上限後のUXが「再試行」ボタンのみ |

### 根本問題

1. **原因が伝わらない**: ネットワーク系かサーバー系かユーザー操作ミスか不明
2. **次の行動が分からない**: 「再試行」の意味が曖昧（リロード？ボタン再押し？）
3. **誤った安心感**: "結果は保存されています" は事実と異なる場合がある
4. **飲み会コンテキスト無視**: 5秒以内に解決できない UX は場の空気を壊す

---

## Current Behavior

- エラー発生 → 汎用文言表示 → ユーザーが自力で対処

---

## Expected Behavior

- **ネットワークエラー**: 「通信が途切れました。Wi-Fi または電波を確認して、もう一度お試しください。」+ 「再試行する」ボタン
- **スピン失敗（サーバーエラー）**: 「スピンに失敗しました。もう一度ボタンを押してください。」+ 自動で waiting フェーズに戻る
- **ルーム読み込み失敗**: 「ルームの取得に失敗しました。」→ 自動リトライ（3秒後に1回）→ 失敗なら「再読み込み」ボタン
- **ChunkLoadError**: 上限超過後は「ページを完全に更新する（Ctrl+Shift+R）」の説明を追加
- **"結果は保存されています" の表記を削除**: 誤った安心感を与えないこと

---

## Scope

- `app/error.tsx` — エラー境界のメッセージとリカバリーUX改善
- `app/room/[code]/error.tsx` — 同上
- `app/room/[code]/play/use-spin.ts` — `spinError` のメッセージを状況別に設定
- `app/room/[code]/play/use-room-sync.ts` — フェッチ失敗時の自動リトライ（最大1回）追加

---

## Root Cause Hypothesis

エラーメッセージは実装初期に「とりあえず動く」で設定した汎用文言のまま。
ユーザーリサーチや飲み会コンテキストでのテストが行われていない。
エラーが「ユーザーが解決できる問題か否か」の分類もされていない。

---

## Proposed Fix

### A. エラータイプ別メッセージ分類

```typescript
// use-spin.ts

// Before
setSpinError("スピンに失敗しました")

// After (原因別)
if (status === 409) {
  setSpinError("既にスピンが進行中です。少しお待ちください。")
} else if (status >= 500) {
  setSpinError("サーバーエラーが発生しました。もう一度お試しください。")
} else if (!navigator.onLine) {
  setSpinError("インターネット接続を確認してください。")
} else {
  setSpinError("スピンに失敗しました。もう一度お試しください。")
}
```

### B. ネットワークエラー自動リトライ

```typescript
// use-room-sync.ts

const fetchRoomWithRetry = async (retries = 1) => {
  try {
    await fetchRoom()
  } catch {
    if (retries > 0) {
      await new Promise(r => setTimeout(r, 3000))
      fetchRoomWithRetry(retries - 1)
    } else {
      setError("ルームの取得に失敗しました。")
    }
  }
}
```

### C. エラー境界のメッセージ改善

```tsx
// app/room/[code]/error.tsx

// Before
"ルームの読み込みに失敗しました。ルーレットの結果は保存されています。再試行してください。"

// After
"ルームの読み込みに失敗しました。"
"通信環境を確認して、もう一度お試しください。"
// ← "ルーレットの結果は保存されています。" を削除
```

## Risk / Notes
- 自動リトライは無限ループにならないよう最大1〜2回に制限すること
- HTTP ステータスコードによる分岐は `/api/spin` route のレスポンス設計と一致させること
- ChunkLoadError の CTRL+SHIFT+R 説明は日本語で分かりやすく（モバイルユーザーには「ブラウザを完全に閉じて再度開く」と案内）
