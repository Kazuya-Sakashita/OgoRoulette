# ISSUE-078: 初回シェア前の「公開名確認」ボトムシート実装

## ステータス

## 優先度
High

## カテゴリ
Frontend / UX / Privacy

## 概要
ログインユーザーが初めてシェアアクションを起こす前に、
「この名前でシェアしますか？」の確認ボトムシートを1回だけ表示する。

## UI 仕様

```
┌────────────────────────────────┐
│ 📢 SNS に公開される名前の確認   │
│                                │
│ ルーレット結果や共有テキストに   │
│ 表示される名前を確認してください │
│                                │
│  ┌──────────────────────────┐  │
│  │  ユーザー3a2f         ✏  │  │
│  └──────────────────────────┘  │
│                                │
│  ※ この名前がSNSに公開されます  │
│                                │
│  [変更する]  [この名前でシェア]  │
└────────────────────────────────┘
```

## 表示条件
- ログイン済みユーザーのみ（ゲストは対象外）
- `profile.displayNameConfirmedAt === null` の場合のみ表示
- シェアボタン（X / LINE / コピー / Web Share API）を押した時にインターセプト

## 確認後の動作
1. 「この名前でシェア」→ `display_name_confirmed_at` を記録して即シェア
2. 「変更する」→ display_name 入力フォームを展開 → 保存後にシェア
3. 以降は確認なしで直接シェア

## 実装メモ
- `components/display-name-confirm-sheet.tsx` として新規作成
- `useDisplayNameConfirm()` hook で確認済み状態を管理
- ISSUE-079 のプロフィール編集とロジックを共有する
