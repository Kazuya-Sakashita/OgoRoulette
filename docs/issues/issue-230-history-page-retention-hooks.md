# ISSUE-230: UX改善(P1) — 履歴ページのリテンション強化：空状態からグループ再利用へ

## ステータス
✅ 完了 2026-04-07

## 優先度
**P1 / High** — AARRR-Retention の構造的ボトルネック。履歴ページが「見るだけ」で再利用を促せていない

## カテゴリ
UX / Retention / History / Re-engagement

## 対象スコア
AARRR-Retention: +1 / HEART-Adoption: +0.5 → 総合 +1.5点

---

## Summary

`/history` ページは「過去のルーレット履歴を見る」以上のことができない。
ゲストユーザーには空状態 + ログイン促進のみ。ログインユーザーにも履歴リストがあるだけで、
「次の飲み会でこのメンバーをすぐ使う」というアクションへの導線がない。

---

## Background

### 現状の /history ページ

**ゲスト（未ログイン）:**
```
👑 ログインで履歴をクラウド保存
端末をまたいでいつでも確認できます
[ログインする]

✨ ルーレットを回すとここに履歴が表示されます
```
→ 「ルーレットを回せ」というCTAがない。「今すぐ回す」への誘導がない。

**ログインユーザー（履歴あり）:**
```
セッション一覧が表示されるだけ
```
→ 「このメンバーでまた回す」ボタンがない（グループ保存からは可能だが、履歴からは無理）

### リテンション構造の問題

飲み会は週次・月次で繰り返し開催される。
同じグループで何度も使う可能性が高いのに、
「前回のメンバーで新しいルームを作る」という1クリックフローが存在しない。

---

## Current Behavior

1. /history を開く
2. ゲストは空状態 + ログインCTAのみ
3. ログイン済みは過去セッションのリストのみ
4. 「また使う」のアクションができない

---

## Expected Behavior

### A. ゲスト向け空状態改善

```
現在のメッセージ:
「ルーレットを回すとここに履歴が表示されます」

改善後:
🎯 次の飲み会も、ルーレットで決めよう

  グループを保存すると
  毎回メンバーを入力せずにすぐ回せます

  [ルームを作る] ← 主CTA
  [使い方を見る] ← サブCTA
```

### B. 履歴エントリから「また回す」

セッション履歴の各アイテムに:
```
[山田さんが奢り — 2026-04-07]
参加者: 田中, 山田, 佐藤, 鈴木 (4人)

[→ このメンバーで新しいルームを作る]  ← 追加
```

「→ このメンバーで新しいルームを作る」を押すと:
- 参加者リストを保持してルーム作成ページへ遷移（または直接ルーム作成）
- または「グループとして保存してからルームを作る」2ステップ

### C. ログインユーザーの空状態（初回スピン前）

```
🎯 初めてのルーレット

[ルームを作る]     [1人で試す]
```

---

## Scope

- `app/history/page.tsx` — 空状態のメッセージとCTA改善、セッション履歴からの「また回す」追加
- `app/history/[id]/page.tsx` — セッション詳細からの「このメンバーで再利用」CTA

---

## Root Cause Hypothesis

履歴ページは「記録を見る」という情報系ページとして設計されており、
「次のアクションへ誘導する」変換系ページとして設計されていない。
リテンションを上げるには記録を「再利用の起点」にする必要がある。

---

## Proposed Fix

### app/history/page.tsx — 空状態

```tsx
// ゲスト向け（未ログイン）
{!sessions.length && (
  <div className="text-center py-12">
    <div className="text-4xl mb-4">🎯</div>
    <h2 className="text-lg font-bold text-foreground mb-2">
      次の飲み会も、ルーレットで決めよう
    </h2>
    <p className="text-sm text-muted-foreground mb-6">
      グループを保存すると、毎回メンバーを入力せずにすぐ回せます。
    </p>
    <Button asChild className="bg-gradient-accent text-white">
      <Link href="/room/create">ルームを作る</Link>
    </Button>
  </div>
)}
```

### 履歴リストの「また回す」ボタン

```tsx
// セッションカード内
<Link
  href={`/room/create?members=${encodeURIComponent(session.participants.join(","))}`}
  className="text-xs text-primary hover:underline mt-2 block"
>
  → このメンバーで新しいルームを作る
</Link>
```

※ ルーム作成ページ側で `?members=` パラメータを受け取り、参加者を事前入力する機能が必要
