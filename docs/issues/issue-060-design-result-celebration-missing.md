# ISSUE-060: スピン後の当選発表に祝祭感がなく、感情的クライマックスを逃している

## ステータス
✅ 完了（Phase 1）

## 優先度
**Critical**

## カテゴリ
Design / UX / Core Feature

## 概要
ルーレットが止まった後（当選発表）の画面演出が乏しく、「誰が奢るか決まった！」という感情的ピークが体験できない。スピン後はホイールが止まるだけで、パーティクル・音・大きなアニメーションがなく、インパクトある瞬間が訪れない。この瞬間はアプリのハイライトであり、SNS でのシェア動機にもなる重要なポイント。

## 問題の詳細

### 現状の当選発表フロー
1. ホイールがスローダウン
2. 止まる
3. WinnerCard（当選者名カード）が表示される
4. 以上

### 欠けている演出
- confetti（紙吹雪）アニメーション
- 当選セグメントのハイライト（グロー・パルス）
- 「🎉 当たった！」的な画面全体の演出
- 結果の SNS シェアへの自然な誘導

### WinnerCard の問題
- カードが表示されるが、当選者名のフォントサイズが控えめ
- 背景演出がなく、テキストが淡々と表示されるだけ
- 「おごりの金額」は表示されるが、「このメンバーの中からこの人が選ばれた」という文脈演出がない

## 改善案

### Phase 1: 演出追加（工数小〜中）
```bash
bun add canvas-confetti
# または react-confetti
```

```typescript
// スピン完了時
import confetti from 'canvas-confetti'

const handleSpinComplete = (winner: Member) => {
  confetti({
    particleCount: 150,
    spread: 70,
    origin: { y: 0.6 }
  })
}
```

### Phase 2: WinnerCard の強化
- 当選者名を極大フォント（text-5xl 以上）で表示
- 当選者のアバター色をアニメーションで強調
- 「〇〇さんが今日のおごり！」のコピーを追加

### Phase 3: シェア導線
- 結果画面に「X でシェア」ボタンを追加
- シェアテキスト: 「OgoRoulette で今日のおごりが決まりました 🎰 [ルーム名] の勝者は〇〇さん！」

## 影響範囲
- `components/winner-card.tsx`
- `app/room/[code]/play/page.tsx`（スピン完了時のトリガー）

## 優先理由
「アプリを使って楽しかった」という記憶は当選発表の瞬間に形成される。ここが地味なままだと、口コミ・シェアが生まれず、ユーザー獲得が停滞する。
