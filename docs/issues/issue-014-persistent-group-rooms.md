# [ISSUE-014] 常設グループ機能（永続ルーム）がなく継続利用の導線が弱い

## 🧩 概要

現在のルームは単発使い捨て設計（`expiresAt` あり）のため、毎回同じメンバーで飲む固定グループが「このルームでまた使おう」と思えない。`UserGroup`（グループ保存）機能はあるが、使うたびに新しいルームを作り直す手間がある。「常設グループ」として永続するルームを持てる機能を追加することで、継続利用率を向上させる。

## 🚨 背景 / なぜ問題か

**現在のユーザーフロー（毎回）:**
1. ホームを開く
2. 「ルームを作成」→ メンバー追加（または保存グループをロード）
3. QR を共有してメンバーを集める
4. SPIN
5. ルームは COMPLETED になり再利用不可（または期限切れ）

**問題:**
- 毎回メンバーが QR スキャンし直す必要がある
- 同じグループが「このアプリをブックマーク」するきっかけがない
- 競合アプリがあれば次回は別のアプリを使う可能性がある
- 課金の動機が生まれない（常設ルームをプレミアム機能にできる）

## 🎯 目的

同じグループが繰り返し使えるルームを作れるようにする。「会社の飲み会ルーム」「大学のサークル飲み会ルーム」として長期間使い続けるための機能。

## 🔍 影響範囲

- **対象機能:** ルーム作成・管理
- **対象画面:** `/room/create`、`/home`、`/room/[code]`
- **対象コンポーネント:**
  - `app/room/create/page.tsx`
  - `prisma/schema.prisma`（Room モデルの `expiresAt` 制御）
  - `app/api/rooms/route.ts`（POST）

## 🛠 修正方針

**DB 変更:**

```prisma
model Room {
  // ...
  expiresAt     DateTime?  // null = 無期限（常設ルーム）
  isPersistent  Boolean    @default(false)  // 常設グループフラグ
  maxSpins      Int?       // null = 無制限
}
```

**API 変更 (`POST /api/rooms`):**

```tsx
// 認証ユーザーのみ常設ルームを作成可能
if (body.isPersistent && !user) {
  return NextResponse.json({ error: "常設ルームはログインが必要です" }, { status: 401 })
}

// 常設ルームは expiresAt = null
const expiresAt = body.isPersistent ? null : new Date(Date.now() + 24 * 60 * 60 * 1000 * 7)
```

**UI 変更（ルーム作成画面）:**

- 「常設グループとして保存」トグルを追加（ログイン済みユーザーのみ表示）
- 常設ルームは COMPLETED 後も `room.status = WAITING` にリセットされる（`reset` API）

**常設ルーム一覧（ホーム画面）:**

- 認証ユーザーのホームに「マイグループ」セクションを追加
- 過去に作った常設ルームを一覧表示
- ワンタップでそのルームのプレイページに移動

**マネタイズ接続:**

- 常設グループ数の上限をフリー 3 個・プレミアム無制限に設定
- プレミアム機能への導線として機能する

## ⚠️ リスク / 副作用

- `expiresAt = null` のルームが増えると DB のデータ量が増加する。定期クリーンアップ（最終使用から N 日後に削除）が必要
- 常設ルームは COMPLETED → WAITING のリセット後も同じ `inviteCode` を使い続けるため、古い参加者も招待なしにアクセスできる。メンバー管理機能が必要

- [ ] 常設ルームは期限切れにならない

- [ ] ホーム画面に常設ルーム一覧が表示される

## 🧪 テスト観点

**手動確認:**
1. 常設ルームを作成 → SPIN → COMPLETED → 「もう一回！」→ 同じ inviteCode で再 SPIN できる
2. 非ログインユーザーが常設ルーム作成を試みる → エラーが表示される
3. フリープランのユーザーが 4 個目の常設ルームを作成しようとする → 制限エラーが表示される（プレミアム施策との連携）
