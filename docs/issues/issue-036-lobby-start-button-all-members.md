# ISSUE-036: ルームロビーの「ルーレットを回す」ボタンが非オーナーメンバーにも表示される

## 概要

`/room/[code]` のロビーページで「ルーレットを回す」ボタンが全メンバーに表示されるが、プレイページではオーナー（ホスト）しか SPIN ボタンを押せない。メンバーはボタンを押しても何もできない状態でプレイページに遷移してしまう。

## 背景

ロビーページはホスト・メンバー共通の待合室。「ルーレットを回す」ボタンは全員に同じ UI で表示されている。プレイページに遷移した後にオーナー判定が行われ、非オーナーには SPIN ボタンが disabled になる。

## 問題点

- 現在何が起きているか: 非オーナーメンバーがロビーで「ルーレットを回す」を押すとプレイページに遷移する。しかしそこで SPIN ボタンを押しても何も起きない（disabled）
- ユーザー影響: 初見ユーザーが「壊れてる」と思い離脱する
- 技術的影響: 機能的な問題はなし。UX の問題

## 原因

`app/room/[code]/page.tsx` のボタン表示ロジックにオーナー判定が含まれていない。

## 修正方針

以下のいずれかで対応：
1. ホスト（`room.members.find(m => m.isHost)` のプロフィールが自分のもの）かどうかを判定し、メンバーには「ルーレットを見る」と表示を変える
2. メンバーには「参加中 — ホストがルーレットを始めると自動で始まります」というメッセージを表示し、ボタンは非表示にする

Supabase ユーザー情報をクライアントで取得して比較する必要がある。

## タスク

- [ ] ロビーページで現在ユーザーの `profileId` を取得
- [ ] `room.members.find(m => m.profileId === currentUser?.id && m.isHost)` でオーナー判定
- [ ] 非オーナー向けの待機 UI を実装（「ホストがルーレットを開始するまでお待ちください」）
- [ ] ゲストホスト（`profileId=null`）の場合も localStorage の `ogoroulette_host_rooms` で判定
- [ ] 動作確認

## 受け入れ条件

- オーナーだけに「ルーレットを回す」ボタンが表示される
- 非オーナーには待機メッセージが表示される
- ページリロード後も正しく判定される
