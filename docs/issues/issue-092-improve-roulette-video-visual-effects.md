No

## 依存
ISSUE-091 完了後に実施すること

## 概要

ISSUE-091 で「録画タイミング」の構造的欠陥を修正した後、
動画の「視覚的な情報伝達力」を強化する。

SNS でシェアされた動画が音なしでも「誰が奢り担当になったか」「盛り上がっている場か」が
一目で伝わるようにする。

---

## 背景

現状の RecordingCanvas の課題:

1. **冒頭にコンテキストがない**: 動画を見た人が「誰がいるか」「何の動画か」を理解する前に
   ホイールが回り始める。SNS で見た第三者は置いてけぼりになる。

2. **当選名テキストが小さい**: 540×960px のキャンバスに対して、
   当選名の表示サイズが不十分。無音視聴者に「誰が選ばれたか」が瞬時に伝わらない。

3. **オチが弱い**: リアクションテキスト（"ごちそうさまです！" など）はランダム選択だが、
   状況（人数・金額）に応じたバリエーションがなく汎用的すぎる。

---

## 改善内容

### 改善A: 動画冒頭に参加者リスト表示（イントロフェーズ）

`RecordingCanvas` に `"intro"` フェーズを追加する（録画中最初の1.5秒）。

**描画内容:**
```
┌─────────────────────────────┐
│  🎰 OgoRoulette             │  ← ロゴ
│                             │
│  今日の参加者               │  ← タイトル
│  • Aさん                   │
│  • Bさん                   │  ← 参加者リスト（最大8名）
│  • Cさん                   │
│                             │
│    〜 誰が奢る？ 〜         │  ← サブタイトル
└─────────────────────────────┘
```

**フロー変更:**
```
"intro" (1.5秒) → "countdown" (3秒) → "spinning" (4.5秒) → "reveal" (4.5秒)
                    ↑ここから既存
```

**実装箇所:**
- `components/recording-canvas.tsx`: "intro" フェーズの描画処理を追加
- `lib/use-video-recorder.ts`: 録画開始を "intro" フェーズに変更 (ISSUE-091で "countdown" に変更済みのものをさらに前倒し)
- `app/room/[code]/play/page.tsx`: RecordingPhase に "intro" を追加、参加者データを RecordingCanvas に渡す

---

### 改善B: 当選名テキストの視覚インパクト強化

`recording-canvas.tsx` の reveal フェーズ描画を改善する。

**Before:**
- 当選名フォント: 40px → (progress で拡大)
- グロウシャドウ: 通常

**After:**
- 当選名フォント: 60px（540px幅に対してより大きく）
- 太字フォントに変更: `"bold 60px system-ui"`
- グロウシャドウを2重にする: `blur: 30px` + `blur: 60px`
- 文字の下に「帯（矩形）」を置いて視認性を上げる

---

### 改善C: 状況連動型オチテキスト

現状の `REACTION_TEXTS` はランダム配列。
参加者情報・金額情報をもとに状況に応じたテキストを選ぶ。

**バリエーション例:**

```typescript
// 人数ベース
2人: ["一騎討ち...", "50%の悲劇！", "2択で負けた..."]
3〜5人: ["運命のルーレット！", "ドンマイ...😅"]
6人以上: ["6分の1の確率...", "大人数の中から選ばれし者"]

// 金額ベース（totalBill がある場合）
¥10,000以下: ["ごちそうさまです！"]
¥10,000〜¥30,000: ["太っ腹！！", "大盤振る舞い！"]
¥30,000以上: ["え...本当に...？", "神すぎる...🙏"]

// デフォルト（金額なし）
["ごちそうさまです！", "次は私の番かも...", "また誘ってください🍽️"]
```

## 受け入れ条件

- 動画冒頭で参加者リストが表示される（音なしでも「誰の動画か」わかる）
- 当選名が大きく表示され、SNSのタイムラインで一目でわかる
- オチテキストが状況に応じていて、SNSで「クスッ」とする
- 動画総尺が 13〜15 秒であること（長すぎない）
- 参加者が1〜12人の場合に正常に動作する

```typescript
  // ...
}
```

`participants` はすでに渡されているため、イントロフェーズで使える。
