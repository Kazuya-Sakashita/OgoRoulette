# ISSUE-059: プレイページの情報密度が高く、ホイールまでのスクロールが必要

## ステータス
✅ 完了 — 2026-04-06

## 優先度
**High**

## カテゴリ
Design / UX

## 概要
プレイページ（`/room/[code]/play`）はヘッダー・有効期限バナー・メンバーリスト・金額/人数設定が縦に積み重なり、ルーレットホイールが最初から全体で見えないケースがある。「押したい・回したい」という衝動を持つユーザーに対し、すぐにホイールを見せてスピンさせることが最優先であるべき。

## 問題の詳細

### 情報の優先順位（現状）
1. ナビゲーションヘッダー（戻る・ルーム名・メニュー）
2. 有効期限バナー（黄色の警告色）
3. 参加メンバーのアバター一覧
4. 設定行（奢り金額・最大人数）
5. **ルーレットホイール** ← ユーザーが来た目的
6. SPIN ボタン

機能的には正しいが、ユーザーの動線（「回したい」）に対して妨げが多い。

### 有効期限バナーの常時表示
- 期限が数日以上ある場合でも amber 色（警告色）のバナーが常に表示される
- 警告色は「今すぐ対処が必要」を意味するが、通常状態でも同じ色が出ることで慣れ（警告疲れ）が起きる
- バナーが存在するだけで画面の色温度が上がり、落ち着いた体験を阻害する

### メンバーリストの冗長性
- プレイページで「誰が参加しているか」は既知情報（ロビーで確認済み）
- アバター一覧がホイールのセグメントと重複した情報を持つ

## 改善案

### 1. 有効期限バナーの条件付き表示
```typescript
// 残り3日以内のみ警告色で表示、それ以上は非表示 or info色
const daysLeft = getDaysUntilExpiry(room.expiresAt)
if (daysLeft > 3) return null  // バナー非表示
```

### 2. メンバーリストの折りたたみ
- デフォルトは折りたたみ（「3名参加中 ▼」のみ表示）
- タップで展開
- ホイールのセグメントで誰が参加しているかは視覚的に確認できるため冗長性が減る

### 3. 設定行のセカンダリ化
- 奢り金額・最大人数は小さいテキストかアイコンボタンに縮小
- 設定変更はモーダルまたはホワイルシートで行う

## 影響範囲
- `app/room/[code]/play/page.tsx`
- 有効期限バナーコンポーネント（条件付きレンダリング）
