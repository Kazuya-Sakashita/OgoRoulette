# ISSUE-127: ゲスト参加者向けログイン誘導UIをplayページに追加する

## 概要

ゲストとしてルームに参加したユーザーに対し、playページのヘッダーで
ログインを促すインラインCTAを表示する。
公開名の設定メリットを伝えつつ、アカウント登録を自然に誘導する。

---

## 背景

- ゲストユーザーは `nickname`（一時的な名前）のみで参加しており、公開名を持たない
- 次回参加時に名前を再入力する必要があり、継続利用のハードルになっている
- ゲームの「最高潮の瞬間」（playページ）でのログイン誘導は転換率が高い
- ヘッダーのサブテキストとして表示することで邪魔にならずに訴求できる

---

## 修正内容

### `app/room/[code]/play/page.tsx`

```tsx
{!currentUser && !isOwner && (
  <p className="text-xs text-muted-foreground">
    ゲスト参加中 ·{" "}
    <Link
      href="/auth/login"
      className="text-primary hover:underline"
    >
      ログインして公開名を設定
    </Link>
  </p>
)}
```

- `!currentUser`: 未ログインユーザーのみ表示（ログイン済みには非表示）
- `!isOwner`: ホストには表示しない（ホストは既にログイン済みのはず）
- シンプルなインラインリンクで邪魔にならないデザイン

---

## 影響範囲

- `app/room/[code]/play/page.tsx`
- ゲストユーザーのアカウント登録率向上
- ログインページへの自然な導線
