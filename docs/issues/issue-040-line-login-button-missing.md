# ISSUE-040: LINE ログインボタンがウェルカムページに存在しない

## 概要

LINE OAuth のバックエンドフロー（`/api/auth/line/start`、`/api/auth/line/callback`）は完全に実装されているが、ウェルカムページ（`app/page.tsx`）に LINE ログインボタンが存在しない。ユーザーは LINE でログインする手段がない。

## 背景

LINE ログイン機能の実装（ISSUE-038 修正含む）がバックエンドに存在し、LINE OAuth の CSRF 対策・トークン交換・プロフィール取得・セッション確立まで一通り動作する状態にある。しかしフロントエンドのエントリーポイントとなるボタンが UI に存在しない。

## 問題点

### 現在何が起きているか
- `app/page.tsx` には「Googleでログイン」と「まず試してみる（ゲスト）」の 2 つの選択肢のみ
- LINE ログインのエントリーポイントがどのページにも存在しない
- `/api/auth/line/start` は直接叩けば動作するが、ユーザーには見えない

### ユーザー影響
- LINE ユーザーがアカウント連携できない
- Google アカウントを持たないユーザーがログイン機能を利用できない
- ゲストモードのみ使えるため、グループ保存・履歴機能が利用不能

### 技術的影響
- LINE OAuth の実装コードが本番で実質デッドコードになる
- `/api/auth/line/start` への rate limit（5回/10分）が意味をなさない

## 原因

ウェルカムページに LINE ログインボタンを追加する実装が漏れていた可能性が高い。または意図的に後回しにされていた。

## 修正方針

`app/page.tsx` に LINE ログインボタンを追加する。

```tsx
// LINE ログイン開始: /api/auth/line/start にリダイレクト
const handleLineLogin = () => {
  router.push('/api/auth/line/start')
}
```

ボタンデザインは LINE ブランドカラー（#06C755）を使用し、Google ボタンと同じ高さ・幅で配置する。

代替案: LINE ログインを当面提供しない場合は、`/api/auth/line/*` のバックエンドコードを削除または非公開化する（エンドポイントが存在していると attack surface になる）。
