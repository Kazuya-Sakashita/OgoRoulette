# ISSUE-145: WinnerCardのCTA順序改善 + /join リダイレクト追加

## 概要

gstack 再評価スコア 81/100 を受けて、Engagement・Retention・Task Success の3軸を改善する。

---

## 背景

- WinnerCard のバイラルCTA「友達に教える」が Guest→Login CTAの**下**に配置されており、ゲストユーザーが気づかずに離脱していた
- Guest→Login CTA のテキストが汎用的すぎて行動動機が弱かった（「次回もすぐに使える」）
- `/join`（コードなし）にアクセスすると 404 になり、QR参加フローへの入り口が失われていた

---

## 変更内容

### 変更1: バイラルCTA を Guest→Login CTA より上に移動

**該当箇所:** `components/winner-card.tsx`

**変更前の順序:**
```
奢りランキング
↓ Guest→Login CTA（ログイン訴求）
↓ バイラルCTA（友達に教える）← 下すぎて届かない
↓ 次のアクション
```

**変更後の順序:**
```
奢りランキング
↓ バイラルCTA（友達に教える）← 全ユーザーが到達
↓ Guest→Login CTA（ゲストのみ表示）
↓ 次のアクション
```

**効果:** ログインユーザーもゲストも「友達に教える」ボタンを確実に目にする

---

### 変更2: Guest→Login CTA のテキストをタイムリーに変更

**該当箇所:** `components/winner-card.tsx`

**変更前:**
```
タイトル: 次回もすぐに使える
本文: ログインするとメンバー保存・履歴閲覧ができます
```

**変更後:**
```
タイトル: この履歴、端末が変わると消えます
本文: ログインしてクラウドに保存しておこう
```

**効果:** スピン直後に「この瞬間のデータが消えるかもしれない」という具体的な危機感を訴求。抽象的な機能説明より転換率が高い。

---

### 変更3: `/join` → `/scan` リダイレクト追加

**該当箇所:** `app/join/page.tsx`（新規作成）

**変更前:** `/join` にアクセスすると 404

**変更後:** `/scan`（QR スキャン＆コード入力ページ）へ即時リダイレクト

```typescript
import { redirect } from "next/navigation"

export default function JoinPage() {
  redirect("/scan")
}
```

**備考:** `/join/[code]` の動的ルートは既存のまま変更なし。`/join`（コードなし）のみ対象。
