# WinnerCard Phase B の演出・デザイン改善

## 概要

WinnerCard は Phase A（シネマティック演出 3.2s）と Phase B（詳細シート）の2段構成だが、Phase B は機能的なリスト画面に急落しており、Phase A で高まった感情が一気に冷める。Phase B を「演出の続き」として体験できるよう改善する。

## 背景

WinnerCard Phase A は以下の要素が組み合わさり、ハッカソン作品として突出した演出クオリティを持つ:
- 300ms: Crown アイコンのフェードイン
- 650ms: 当選者名の大きなテキスト
- 1150ms: 反応テキスト（「ごちそうさまです！」等）
- 1500ms: インスタントシェアボタン
- 3200ms: Phase B へ自動遷移

しかし Phase B は「支払い内訳・ランキング・各種ボタン」が並ぶ機能的な画面。Phase A の全画面シネマティックから急に「ビジネス画面」に変わり、感情の落差が大きい。

## 現状の問題

1. **Phase A → Phase B の遷移アニメーションが弱い**: フェードアウト/インだけで「続き」感がない
2. **Phase B が情報過多**: シェアボタン・ランキング・保存・リスピン・ログイン CTA が並列で視認性が低い
3. **金額表示が埋もれている**: 最も知りたい「割り勘金額」がリストの中に埋もれている
4. **ゲスト向けのログイン CTA がタイミング悪い**: 感情の高潮のすぐ後にサインアップを促される
5. **背景が Phase A から変わりすぎる**: 全画面グラデーションから背景色カードに切り替わるため「映画が終わった感」がある

## 目的

- Phase A の感情的高揚を Phase B まで維持する
- 最も重要な情報（当選者・金額）を Phase B でも前面に出す
- アクションの優先順位を整理し、ユーザーが迷わず次の行動を取れるようにする
- ログイン CTA を「感動を遮らないタイミング」に移動する

## 対応内容

### 1. Phase A → Phase B の遷移改善

- 遷移をスライドアップ（`y: 100% → 0%`）にして「シートが引き上がってくる」感覚を作る
- 遷移時間: 0.5s（現在より長く、ゆっくりとした出現）
- Phase B の背景に Phase A の当選者カラーを薄く残す（急な色変化を和らげる）

### 2. 金額表示の強調

Phase B 最上部に当選者名 + 金額を大きく表示:
```
[Crown] たろうさんが奢り！
────────────────────
合計 ¥10,000 → 割り勘 ¥2,500/人
```

### 3. アクションボタンの整理

Primary（1つだけ）: 「シェアする」（最も重要）
Secondary（2つ）: 「もう一度回す」「グループを保存」
tertiary（下部）: ログイン CTA・履歴

### 4. ゲスト向けログイン CTA のタイミング変更

現在: Phase B の上部付近（感情の高潮の直後）
改善: Phase B の末尾（スクロールで初めて見える位置）
コピー変更: 「ログインして履歴を残す」（今の「ログイン」よりも価値提案が明確）

### 5. ランキングの扱い

現在: Phase B で全セッション履歴のランキングを表示
改善: 折りたたみ式（「過去の奢り記録 ▼」として初期は閉じておく）
→ 金額・シェア・リスピンが先に見え、ランキングは興味のある人だけ開く

## 完了条件

- [ ] Phase A → Phase B の遷移がスライドアップになっている
- [ ] Phase B 最上部に当選者名と割り勘金額が大きく表示される
- [ ] シェアボタンが明確に Primary アクションになっている
- [ ] ゲスト向けログイン CTA が Phase B の末尾に移動している
- [ ] ランキングが折りたたみ式になっている
- [ ] モバイル（iPhone SE〜iPhone Pro Max）で崩れない

## 優先度

**Recommended** — Phase A の演出投資が Phase B で無駄になっている。シェア意欲に直結する改善。

## 期待効果

- Visual スコア: 72 → 76（+4）
- UX スコア: 65 → 68（+3）
- シェアボタンのタップ率向上（primary CTA が明確になることで）
- ゲスト → ログイン CVR の改善（CTA タイミングを感情の余韻後に変更）
