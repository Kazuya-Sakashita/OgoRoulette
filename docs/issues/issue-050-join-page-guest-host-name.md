# ISSUE-050: 招待ページでゲストホストの名前が「ゲストさん」になる

## ステータス
🔴 未対応

## 概要
ゲスト（未ログイン）がルームを作成した場合、招待ページ（`/join/[code]`）の招待文が
「**ゲストさん**があなたを招待しています」と表示される。
実際のホスト名（例: "田中"）は `members[].nickname` に存在するにもかかわらず参照されていない。

## 再現手順
1. `/room/create` でゲストとしてニックネーム（例: "田中"）でルームを作成
2. 発行された招待コードで `/join/[code]` を別ブラウザで開く
3. 「**ゲストさん**があなたを招待しています」と表示される（"田中さん" ではない）

## 原因
`app/join/[code]/page.tsx:187`:
```tsx
{room?.owner?.name ?? "ゲスト"}さんがあなたを招待しています
```
- ゲスト作成ルームでは `room.owner` は `null`（`ownerId` が NULL）
- フォールバックが固定文字列 `"ゲスト"` になっている
- ホスト名は `room.members.find(m => m.isHost)?.nickname` で取得可能

## 修正方法
```tsx
// Before
{room?.owner?.name ?? "ゲスト"}さんがあなたを招待しています

// After
{room?.owner?.name ?? room?.members.find(m => m.isHost)?.nickname ?? "ゲスト"}さんがあなたを招待しています
```

`/api/rooms/[code]` のレスポンスには既に `members` 配列（`isHost: true` を含む）が含まれているため、
API変更は不要。フロントのみ1行修正。

## 影響
- **ユーザー体験**: 招待を受けた人が「誰に招待されたか分からない」。初見ユーザーが混乱する。
- **信頼性**: ゲストユーザーがほとんどである初期フェーズに特に顕著。
- **重大度**: ゲスト作成ルームの全招待ユーザーに影響（= 大多数のユーザー）
