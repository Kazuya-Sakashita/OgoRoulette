# ISSUE-061: 招待参加ページが素っ気なく、参加へのモチベーションを高められていない

## ステータス
🔴 未対応

## 優先度
**Medium**

## カテゴリ
Design / UX

## 概要
ゲストが招待リンクから来る参加ページ（`/join/[code]` または `/room/[code]/join`）は、現在の参加人数とボタンのみのシンプルな構成。「このルームに参加して何が起きるのか」「今誰がいるのか」「どのくらい楽しそうか」が伝わらず、参加を躊躇させるリスクがある。招待流入は最も重要なユーザー獲得経路であり、ここの転換率は事業指標に直結する。

## 問題の詳細

### 現状の参加ページ要素
- ルーム名（またはコード）
- 現在の参加人数表示
- 「参加する」ボタン

### 欠けている情報・演出
- ルームオーナー（招待した人）の名前・アイコン
- 「〇〇さんがあなたを招待しています」というパーソナライズ文言
- 既に参加しているメンバーのアバター一覧（社会的証明）
- ルーレットのミニプレビュー（何をするアプリかを直感的に理解させる）
- 奢り金額の予告（「今日は 1,000円のランチ代を決めます」）

### デザインの現状
- 背景・アニメーション等のビジュアル要素がなく、静的なフォームに近い
- ゲストが初めてアプリを見る瞬間にもかかわらず、印象に残る要素がない

## 改善案

### コンテンツ追加
```
招待ページの推奨構成:
1. ロゴ / アプリ名（初見ユーザーへのブランド認識）
2. 「[オーナー名] さんがルーレットに招待しています 🎰」
3. ミニルーレットアイコン or アニメーション
4. 参加者アバター一覧（最大5名程度）
5. 金額・設定情報（任意）
6. 「参加する」ボタン（大きく・目立つ色）
7. 「ログインして参加する」サブリンク（アカウント登録促進）
```

### 視覚強化
- ページ背景にブランドカラーのグラデーション or オーブ
- 「参加する」ボタンをプライマリカラー・大型・press-effect 付き

## 影響範囲
- `app/join/[code]/page.tsx`（または相当のルート）
- ルーム情報の API から追加データ取得が必要な場合あり
