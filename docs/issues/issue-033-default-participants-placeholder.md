# ISSUE-033: ホーム画面の初期参加者が ["A", "B", "C", "D"] でプレースホルダー丸出し

## 概要

`app/home/page.tsx` の初期 participants state が `["A", "B", "C", "D"]` に設定されており、初回訪問ユーザーや未ログインユーザーが開いた瞬間にアルファベット文字のルーレットが表示される。プロダクトの第一印象として「作りかけ感」を与える。

## 背景

```typescript
// app/home/page.tsx line 32
const [participants, setParticipants] = useState(["A", "B", "C", "D"])
```

開発用プレースホルダーがそのまま本番に出ている状態。

## 問題点

- 現在何が起きているか: ホーム画面を開くと "A", "B", "C", "D" のルーレットが表示される
- ユーザー影響: 「これはデモ？完成していないアプリ？」という第一印象を与え、初回利用の離脱率が上がる
- 技術的影響: なし

## 修正方針

**オプション A（推奨）:** 意味のある日本語のデモ名に変更する:
```typescript
useState(["田中", "鈴木", "佐藤", "高橋"])
```

**オプション B:** グループが未保存かつ参加者が無編集の場合は空状態（参加者追加の誘導UI）を表示する:
```typescript
useState<string[]>([])
// 0人の場合は「参加者を追加してください」の誘導メッセージ
```

**オプション C:** 絵文字アバターを使った楽しいデモ名にする:
```typescript
useState(["🍺 太郎", "🍕 花子", "🍣 次郎", "🎉 美子"])
```

## タスク

- [ ] `app/home/page.tsx` の初期 participants を意味のある値に変更
- [ ] 空状態の場合の UX を検討・実装（参加者追加の誘導）
- [ ] 動作確認: 初回訪問でリアルなデモ体験ができること

## 受け入れ条件

- "A", "B", "C", "D" が初期値として表示されない
- 初見ユーザーがアプリの使い方を直感的に理解できる

## 優先度

Medium

## デプロイブロッカー

No（機能的問題ではなく印象問題）
