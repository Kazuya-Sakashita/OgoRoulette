# ISSUE-064: ロビーページの QR コードと招待 UI が機能的で、ワクワク感に欠ける

## ステータス
🔴 未対応

## 優先度
**Low**

## カテゴリ
Design / UX

## 概要
ロビーページ（`/room/[code]/lobby`）の QR コード表示と招待 UI は機能的に正しく動作しているが、「みんなで集まってルーレットをする」というソーシャルな体験のワクワク感を演出できていない。特に QR コードが白背景の正方形として素朴に配置されており、ブランドらしさがない。

## 問題の詳細

### 現状のロビー構成
- ルーム名・コード表示
- QR コード（白背景・枠あり）
- 参加メンバーのアバター一覧
- 有効期限バナー

### 改善余地
1. **QR コードのブランディング**: 中央にアプリロゴを入れた QR コード（多くの QR ライブラリがサポート）
2. **待機演出**: 「メンバーが参加するのを待っています...」というアニメーション付きテキスト（ドット点滅等）
3. **メンバー参加時のフィードバック**: 新メンバーが加わったときのトースト通知（現状 Realtime で更新されるが無音・無演出の可能性）
4. **スピン開始ボタンへの誘導**: 全員が揃ったら「準備完了！スピンへ」のボタンが目立つように変化する

### QR コードの見た目
現状の QR コードは機能的だが:
- アプリのダークテーマに対して白い四角が浮いて見える
- ロゴなしの素の QR コードはブランドを感じさせない

## 改善案

### QR コードへのロゴ埋め込み
```typescript
// qrcode.react などを使う場合
<QRCodeSVG
  value={inviteUrl}
  imageSettings={{
    src: "/images/logo-icon.png",
    height: 24,
    width: 24,
    excavate: true,
  }}
/>
```

### 参加アニメーション
新しいメンバーが追加されたとき:
- アバターがフェードインまたはスケールインで追加される
- 「🎉 [名前] さんが参加しました！」のトースト表示

### 全員揃った時の状態変化
```typescript
const allReady = members.length >= minMembers
// allReady になったら「スピン開始」ボタンがパルスアニメーションで強調
```

## 影響範囲
- `app/room/[code]/lobby/page.tsx`
- QR コードコンポーネント
- メンバーリストコンポーネント（参加時アニメーション）
