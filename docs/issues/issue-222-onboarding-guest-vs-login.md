# ISSUE-222: UX改善(P1) — 初回体験オンボーディング：ゲストモードと価値命題を明確に伝える

## ステータス
📋 未着手

## 優先度
**P1 / High** — Adoption スコア 5/10 が最大の穴。改善で +3〜4点見込み

## カテゴリ
UX / Onboarding / Adoption

## 対象スコア
HEART-Adoption: +2（5→7/10）/ HEART-Happiness: +1 / G-STACK-Clarity: +1 → 総合 +3〜4点

---

## Summary

トップページのデモルーレットが「誰かが当たって喜ぶ」画面で終わり、
OgoRoulette の本質的価値（「飲み会で奢り担当を楽しく決め、金額も割り勘計算する」）が初回体験で伝わらない。
また「ゲストモード」と「ログイン」の違いが説明なく、ユーザーはどちらを選べばよいか迷う。

---

## Background

### 問題1: デモが価値命題を伝えきれていない

`app/page.tsx` の demo spin は名前が当たるだけで終わる。
**OgoRoulette の差別化ポイント（奢り金額計算、ランキング、グループ保存）がデモに出ていない。**

ユーザーは「ただのルーレット」だと思って離脱する可能性が高い。

### 問題2: ゲストモード vs ログインの違いが不明

「まず試してみる（ゲストモード）」ボタンと「Googleでログイン」が並んでいるが、
**両者の違いが明示されていない**。

実際の違い:
- ゲストモード: ログイン不要・履歴なし・グループ保存不可
- ログイン済み: 履歴あり・グループ保存可・永続ルーム作成可

これが分からないユーザーは「とりあえずゲストで試す」か「ログインしないと使えないのか？」と混乱する。

### 問題3: 初回SPIN後に次の行動が不明

ゲストが `/home` で初めてスピンした後、「ルームを作ってみる」「ログインする」のどちらに進めばよいか導線がない。

---

## Current Behavior

1. `/` でデモスピン → 名前が当たる → "グループを作って試す" CTA
2. ゲストボタンまたはOAuthボタンが並ぶだけ
3. ゲストモードの制限説明なし
4. 初回SPIN後の次アクション導線なし

---

## Expected Behavior

1. デモスピン結果に「¥4,500 → 田中さんが奢り。1人あたり¥1,500」の小さな金額ブレークダウンを表示
2. ゲストボタンにツールチップまたは副テキスト「ログイン不要、履歴は保存されません」を追加
3. 初回SPIN完了後、ポップアップ or バナーで「友達と一緒に使うと楽しい！ルームを作ってみる→」を表示

---

## Scope

- `app/page.tsx` — デモ結果表示に金額ブレークダウン追加（モック値でOK）
- `app/page.tsx` — ゲストボタン副テキスト追加
- `app/home/page.tsx` — 初回SPIN後バナー（`spunOnce` state 活用）

---

## Root Cause Hypothesis

デモは当初「名前だけ当てる」シンプルさ優先で設計された。
金額計算はログイン後機能として後付けされたため、ランディングページに反映されていない。

---

## Proposed Fix

### A. デモ結果画面に金額モックを追加

```tsx
// app/page.tsx — demo winner 表示部分
{demoWinner && (
  <div>
    <p>🎉 {demoWinner}さん が奢り！</p>
    {/* 追加: 仮の金額ブレークダウン */}
    <p className="text-sm text-muted-foreground">
      合計 ¥4,500 → 1人あたり ¥1,500
    </p>
  </div>
)}
```

### B. ゲストボタンに説明を追加

```tsx
<button>
  まず試してみる
  <span className="text-xs text-muted-foreground block">
    ログイン不要・履歴は保存されません
  </span>
</button>
```

### C. 初回SPIN後バナー

```tsx
// app/home/page.tsx — handleSpinComplete 後に表示
{showFirstSpinBanner && (
  <div className="...">
    友達と一緒に使うともっと楽しい！
    <button onClick={() => router.push('/room/create')}>
      ルームを作る →
    </button>
  </div>
)}
```
