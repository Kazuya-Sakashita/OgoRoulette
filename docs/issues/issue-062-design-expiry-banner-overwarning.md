# ISSUE-062: 有効期限バナーが通常時も警告色で表示され、ノイズになっている

## ステータス
✅ 完了 — 段階的カラー実装: 3h以内=赤・3〜24h=黄・24〜72h=青（2026-04-05）

## 優先度
**Medium**

## カテゴリ
Design / UX

## 概要
ロビーページ・プレイページに表示される有効期限バナーが、期限が十分残っている場合でも amber（黄橙）色で表示されている。amber は「注意・警告」を意味する色であり、通常状態での常時表示は「警告疲れ」を引き起こし、本当に期限が迫ったときの注意喚起効果が薄れる。

## 問題の詳細

### 現状
- ルームの有効期限がある場合、常にバナーが表示される
- バナーの色: amber / orange 系（警告色）
- 表示内容: 期限日時

### 問題点
- 期限が7日後でも1日後でも同じ見た目
- 警告色が常に表示されると、ユーザーは無視するようになる（アラート疲れ）
- ページの色温度が上がり、落ち着いたゲーム体験を阻害する
- 「常設グループ」（有効期限なし）との視覚的差異は必要だが、現状の表現が過剰

### 色の意味論
```
🔴 赤  → 緊急・エラー（今すぐ対処）
🟡 黄  → 注意・警告（近く対処が必要）
🔵 青  → 情報（知っておけばOK）
⚪️ グレー → サブ情報（気にしなくていい）
```

期限が1週間以上ある場合はグレーまたは青で「情報」として表示し、3日以内になったら黄、24時間以内は赤に変える、という段階的な色設計が適切。

## 改善案

### 段階的バナー表示
```typescript
function getExpiryBannerVariant(expiresAt: Date): 'none' | 'info' | 'warning' | 'danger' {
  const hoursLeft = (expiresAt.getTime() - Date.now()) / (1000 * 60 * 60)
  if (hoursLeft > 72) return 'none'      // 3日超: 非表示
  if (hoursLeft > 24) return 'info'      // 1〜3日: 情報（青）
  if (hoursLeft > 3)  return 'warning'   // 3〜24時間: 警告（黄）
  return 'danger'                         // 3時間以内: 危険（赤）
}
```

### バナーのスタイル変更
- `variant='none'`: バナー非表示（期限は設定ページでのみ確認可能）
- `variant='info'`: 薄い青 / グレーのインフォバナー
- `variant='warning'`: 現状の amber 色
- `variant='danger'`: 赤系、より大きなテキスト
