# ISSUE-063: ホームページの SPIN ボタンがルーレットと視覚的に分離しており、操作の関係が分かりにくい

## ステータス
🔴 未対応

## 優先度
**Medium**

## カテゴリ
Design / UX

## 概要
ホームページ（`/home`）のルーレットと SPIN ボタンの配置関係が、視覚的に「ルーレットを回すボタン」として伝わりにくい。SPIN ボタンはルーレットホイールの下に配置されているが、ホイールとボタンの間に十分な視覚的結びつきがなく、新規ユーザーがボタンの役割を直感的に理解できない可能性がある。

## 問題の詳細

### 現状のレイアウト
```
[ グループ選択 / ナビゲーション ]
[ ルーレットホイール             ]
[ SPIN ボタン                   ]
[ その他の情報 / 設定            ]
```

### 問題点
- ルーレットとボタンが「セット」として認識されにくい（間に余白がある）
- SPIN ボタンのサイズ・色が他のUIと比較して相対的に控えめ
- 初見ユーザーが「このボタンを押すとルーレットが回る」とすぐ分かるか不明
- ホイール自体にタップ可能な演出がない（ホイールを直接タップしても回らない）

### 競合比較（ベストプラクティス）
多くのルーレットアプリは:
- ホイール中央にタップボタンを配置
- または、ホイール自体をタップ可能にする
- SPIN の文字をホイール内部または矢印ポインター付近に表示

## 改善案

### Option A: SPIN ボタンをホイール内部に配置
```
ホイールの中央に円形ボタンを配置
→ "SPIN" テキスト + 指タップアイコン
→ 「押す = 回る」の直感的な対応関係
```

### Option B: ホイール直下に密着配置
```
ホイールとボタンの間の余白を縮小
ボタンをホイール幅に揃えて配置
プライマリカラーで大きく表示
```

### Option C: ホイール全体をタップ可能にする
```typescript
<div onClick={handleSpin} className="cursor-pointer select-none">
  <RouletteWheel />
</div>
```
シンプルで直感的。ただしモバイルのスクロール競合に注意。

## 推奨
Option A（ホイール中央配置）が最もインパクトがあり、操作の対応関係も明確。工数は中程度。

## 影響範囲
- `app/home/page.tsx`（または相当）
- ルーレットホイールコンポーネント
