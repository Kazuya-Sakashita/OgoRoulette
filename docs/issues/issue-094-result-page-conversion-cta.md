# issue-094: /result ページに「あなたもやる？」CTAを追加し新規ユーザーを獲得する

**ステータス:** ✅ 実装済み（commit: f3dfd44）

## 優先度
High

## デプロイブロッカー
No

---

## 概要

現在の `/result` ページは当選結果を表示するだけ。
SNS でシェアされた URL を開いた非ユーザー（初見の人）に対して、
「自分も使ってみる」という行動に導く CTA が存在しない。

シェアによる流入を「新規ユーザーの獲得」につなげるため、
**シェアされた URL を開いた人がワンタップで自分の飲み会で使い始められる**
導線を追加する。

---

## 背景

現状の `/result` ページの役割:
- 当選者名・色・参加者・金額をOGPとして表示（SNSプレビュー用）
- 「誰が奢り担当になったか」を見せるだけ

**問題**: シェアを見た人（潜在ユーザー）が「面白い！」と思っても、
次にとれるアクションが見当たらない → 離脱。

バイラル係数を上げるには「見た人がユーザーになる」パスが必要。

## 実装方針

### CTA の優先順位

1. **ルームがまだ有効（expiredAt が未来）な場合**: 「このグループに参加する」ボタンを最優先で表示
   - `/join/{code}` にリダイレクト
   - 「Aさんたちがまだ遊んでいます！参加しますか？」というコピー

2. **ルームが期限切れまたは不明の場合**: 「あなたも試してみる？」ボタンを表示
   - `/home` または `/room/create` にリダイレクト
   - 「飲み会の奢り担当を楽しく決めよう」というコピー

### OGP の改善

現状の OGP タイトル/説明を感情訴求型に変更。

```typescript
// app/result/page.tsx
// Before
title: `${winner}さんが奢り担当に！`

// After
title: `👑 ${winner}さんが今日の奢り担当に決定！`
description: `${participants.join("・")} で OgoRoulette を使って奢りを決めました。あなたの飲み会でも試してみませんか？`
```

OGP 画像（og:image）も当選結果をビジュアルで伝えるものに強化。

### URL パラメータの拡張

現状の `/result` URL パラメータ:
```
/result?winner=Aさん&color=%23F97316&...
```

ルームコードも渡せるようにして、ルームがまだ有効か確認できるようにする:
```
/result?winner=Aさん&color=%23F97316&roomCode=ABC123&...
```

`/result` ページで `roomCode` があれば `/api/rooms/{code}` を叩いて
有効かどうかをチェックし、「参加する」ボタンの表示/非表示を決める。

### コピーのトーン

「試してみる」ではなく「体験する」ではなく、
**その飲み会の文脈に引っ張り込む**コピーにする。

```
Aさん、Bさん、Cさん、Dさん が使いました。
あなたの次の飲み会でも？

[今すぐ無料でやってみる]
```

「あなたも」「無料」「今すぐ」が三点セット。

## 受け入れ条件

- シェアされた `/result` URL を開くと CTA が表示される
- ルームが有効な場合は「参加する」ボタンが表示される
- ルームが期限切れの場合は「試してみる」ボタンが表示される

- モバイルで見たとき、CTA がファーストビューに収まる

## 実装メモ

### 優先度の考え方

`/result` ページへの流入は大きく2種類:
1. **ルームが有効（飲み会まだやってる）** → 「参加する」の方が即座の価値
2. **ルームが期限切れ（後日シェアを見た）** → 「次の飲み会で使う」の動機付け

両方に対応することで、リアルタイム流入もアーカイブ流入も取りこぼさない。

### share-service.ts での変更

```typescript
// lib/share-service.ts
// シェアURLにルームコードを追加
const resultUrl = `${APP_URL}/result?winner=${encodeURIComponent(winner)}&color=${encodeURIComponent(color)}&roomCode=${roomCode}&...`
```
