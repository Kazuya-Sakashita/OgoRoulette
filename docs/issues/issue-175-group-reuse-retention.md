# グループ再利用・リテンション施策

## 概要

OgoRoulette は一度使った後に「また使いたい」と思わせる仕組みが不足している。グループ保存・再利用の導線を強化し、「このメンバーでまた回したい」という自然な再訪を促す。

## 背景

現在のグループ機能:
- ホームページでグループを保存できる（localStorage ベース）
- ログインユーザーはサーバー側に保存される
- WinnerCard Phase B に「このメンバーで保存」ボタンがある

しかし:
- グループ保存ボタンは WinnerCard Phase B の多数のアクションの中に埋もれている
- 保存したグループをどう使うかが分かりにくい（ホームに戻ると表示される）
- 「このメンバーで次回はいつ食事する？」という前向きな導線がない
- ゲストユーザーのグループ保存は localStorage のみ → ブラウザを変えると消える
- 「誰が何回奢ったか」のランキングが見えるが、それが「また使おう」というモチベーションに繋がっていない

## 現状の問題

1. グループ保存の導線が埋もれている（Phase B の 5 番目くらいのアクション）
2. 保存したグループを「すぐに使う」動線がない（ホームに戻る必要がある）
3. 「前回このグループで奢ったのは誰？」がすぐに分からない
4. 次回の食事会を意識させる機能がない
5. ゲストの localStorage データはブラウザを変えると失われる

## 目的

- 「このグループでまた回したい」という再訪の動機を作る
- グループ保存の導線を最重要アクションとして位置づける
- 前回の奢り情報（誰が払ったか）を次回の参加前に伝える

## 対応内容

### 1. WinnerCard Phase B のグループ保存を Primary CTA に格上げ

Phase B の最重要アクション（シェアボタンと並べて大きく表示）:
```
[シェアする] [このグループを保存]
```

保存済みの場合: 「保存済み ✓ 次回も使う」

### 2. ホームのグループリストに「前回の奢り担当」を表示

グループカードに:
```
[さくら・たろう・はな・けんた]
前回: たろうさんが奢り（3日前）
```

「前回誰が払ったか」を次回のスピン前に見せることで「今回は別の人に払わせたい」という自然な動機が生まれる。

### 3. 保存グループからワンタップでルーム作成

グループカードに「このメンバーでルームを作る」ボタンを追加。
→ グループメンバーが事前に登録された状態でルームが作成される。

### 4. ゲストユーザーへのアカウント登録誘導（データ保護）

WinnerCard でグループを保存した直後（感情が高いとき）に:
```
「グループを保存しました！
ログインするとブラウザを変えても消えません → ログインして保存」
```

「データが消えるかも」という不安を解消する形で登録を促す。

### 5. 奢り回数ランキングの活用

WinnerCard Phase B のランキングを「次回誰が払うべきか」の参考として打ち出す:
```
奢り回数ランキング:
1位: たろう（3回）← 今回奢り
2位: さくら（2回）
3位: はな（1回）
```
→「はなさんはまだ少ない、次回狙い目！」という会話が生まれる

## 完了条件

- [ ] WinnerCard Phase B の「グループ保存」が大きく目立つ位置にある
- [ ] 保存済みグループカードに「前回の奢り担当者と日時」が表示される
- [ ] グループカードから「このメンバーでルームを作る」が1タップでできる
- [ ] ゲストがグループ保存したときにアカウント登録を促すメッセージが出る
- [ ] 奢り回数ランキングが「次回の参考」として説明されている

## 優先度

**Nice-to-have** — リテンションは長期的な価値だが、まず信頼性・体験改善が先。Phase 3 で対応。
