# ISSUE-030: scan ページのフッターリンク /auth/login が 404

## 概要

`app/scan/page.tsx` のフッターに「新しいルームを作成したい場合はログインしてください」というリンクがあり、`/auth/login` に向いている。しかしこのルートは存在せず、クリックすると 404 になる。

## 背景

認証フローは Google OAuth の場合 `/auth/callback` を経由するため、ユーザーへの入口は通常 `/` (welcome ページ)のログインボタン。`/auth/login` というページは存在しない。

## 問題点

- 現在何が起きているか: `app/scan/page.tsx` のフッター（line 171）から `/auth/login` へのリンクが 404 になる
- ユーザー影響: QR スキャンからログインしようとした新規ユーザーが 404 に当たり離脱
- 技術的影響: なし

## 修正方針

リンク先を `/` に変更する（welcome ページで Google ログインが可能）:

```tsx
// 修正前
<Link href="/auth/login">ログインしてください</Link>
// 修正後
<Link href="/">ログインしてください</Link>
```
