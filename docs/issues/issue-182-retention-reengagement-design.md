# リテンション設計（再訪・再エンゲージメント機能の実装）

## 概要

OgoRoulette の HEART Retention スコア（10/20）・AARRR Retention スコア（10/20）は評価の中で最も低い軸。「今日のおごりを決めた → 次の飲み会まで忘れる」という利用パターンを「次の飲み会の前日に思い出して開く」に変えるための再訪設計を実装する。

## 背景

現状の OgoRoulette は「一度使えばおごりが決まる」という完結型アプリになっており、再訪する理由が乏しい:
- グループを保存しても「保存した」ことを思い出すきっかけがない
- 「前回誰が奢ったか」が記憶に残らないため「今回こそあの人の番では？」という動機が生まれない
- プッシュ通知なし、リマインダーなし、週次ランキングなし

AARRR の North Star Metric（週次ルーム完走数）向上には Retention の改善が不可欠。

## 問題

- グループ保存後の「再利用」フローが存在しない
- ホーム画面に「前回スピンからN日」「前回の奢り担当: ○○さん」が表示されない
- PWAインストール後のプッシュ通知が未実装
- 「また使おう」と思った時にアプリを見つける方法がホーム画面追加以外にない

## 目的

- HEART Retention を 10 → 16 (+6) に改善する
- AARRR Retention を 10 → 16 (+6) に改善する
- 週次ルーム完走数を増加させ North Star Metric を改善する

## 対応内容

### Step 1: ホーム画面のグループ履歴表示強化

グループリストに「前回のスピン: N日前 / 奢り: ○○さん」を表示する。

```typescript
// 表示イメージ
// [飲み会グループ] さくら・たろう・はな・けんた
// 前回: 3日前 | 前回の奢り: さくら
// [ワンタップでルーム作成 →]
```

- `useGroups` フックが持つグループデータに `lastSpinAt` と `lastWinner` を追加
- IndexedDB / localStorage の保存スキーマを拡張する

### Step 2: グループ保存フロー後のコンバージョン改善

スピン結果後（Phase B）でグループを保存した際に「次回はワンタップで始められます」のサクセスメッセージと、ホーム画面追加（PWA）の誘導を表示する。

### Step 3: ホーム画面のアクティブグループ「再開」CTA

グループが保存されているユーザーには、ホーム画面の最上部に:
```
🎰 [飲み会グループ] をもう一度回す
メンバー: さくら・たろう・はな・けんた | 前回3日前
[ルームを作る →]
```
というカードを表示し、過去のグループから即スピン開始できるようにする。

### Step 4: PWAインストール後のプッシュ通知（issue-016との連携）

- Service Worker tombstone（現状）を維持しつつ、プッシュ通知専用のSWを別途追加する設計を検討
- 「次の飲み会の前日（曜日設定）に通知する」オプトイン機能

### Step 5: 「このグループの記録」ページ（将来）

グループIDに紐づいたスピン履歴（誰が何回奢ったか）の一覧ページ。URLで共有できる「おごり記録帳」として機能。
