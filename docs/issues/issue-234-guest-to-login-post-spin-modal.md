## Summary

ゲストユーザーがスピンを完了した直後、当選演出の余韻がある「感情最高潮」のタイミングで「このメンバーを保存する？」モーダルを表示し、ログインへの転換を促す。実装工数2〜3時間に対してゲスト→ログイン転換率の大幅改善が見込める。

---

## Background

gstackテスト（2026-04-15）で確認。ゲストモード（/home）でSPINを2回実行しても、ログインを促す導線が一切表示されない。

HEART評価で Retention が 20→19 に減点された主な原因。当選演出後の感情が高まっているタイミングは、ログイン誘導の最適タイミング。

---

## Current Behavior

- ゲストがスピン完了 → 当選者が表示される
- その後、何のアクションも促されない
- ログインへの誘導は画面下部の小さな「ログイン」リンクのみ

---

## Expected Behavior

### トリガー条件
- ゲストユーザーが **2回目のスピン完了後**（初回は体験させる）
- または1回目でも当選者が表示された後1.5秒後

### モーダル内容

```
┌─────────────────────────────┐
│  🎉 楽しかった？             │
│                             │
│  メンバーを保存すると        │
│  次回もすぐ使えます          │
│                             │
│  [Googleで保存する]          │
│  [LINEで保存する]            │
│                             │
│  [今は保存しない]  ← 小さく  │
└─────────────────────────────┘
```

### 実装方針

1. ゲストスピン回数をlocalStorageに記録
2. 2回目のスピン完了（winner確定）後、1.5秒ディレイでモーダル表示
3. 「今は保存しない」は sessionStorage に記録し、同セッション中は再表示しない
