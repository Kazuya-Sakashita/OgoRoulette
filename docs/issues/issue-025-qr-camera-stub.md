# ISSUE-025: QRスキャンカメラが「実装中」表示でユーザー体験を破壊する

## 概要

`/scan` ページの「QRスキャン」タブに「カメラを起動」ボタンがあるが、タップすると「カメラ機能は実装中です。コード入力をお使いください。」と表示されるだけ。パーティー会場でQRを受け取ったユーザーが「スキャン」を試みて「実装中」に当たると、アプリが壊れていると判断して離脱する。

## 背景

`app/scan/page.tsx` の `handleCameraCapture` は stub 実装で、ユーザーの目の前で「実装中」のメッセージを出す。QRコードを共有した側がQRを見せているのに受け取り側が「スキャン」で詰まるため、パーティーの場で最悪の第一印象を与える。

## 問題点

- 現在何が起きているか: `setCameraNotice("カメラ機能は実装中です。コード入力をお使いください。")` が表示される（`app/scan/page.tsx` line 96-103）
- ユーザー影響: QRスキャン導線がUX上完全に死んでいる。初見ユーザーが最初に試す機能が「壊れている」
- 技術的影響: なし（stub のため副作用なし）

## 原因

```typescript
// app/scan/page.tsx lines 96-103
const handleCameraCapture = () => {
  setCameraNotice("カメラ機能は実装中です。コード入力をお使いください。")
}
```

カメラ機能の実装が間に合っていない状態で UI だけ先行してリリースされた。

## 修正方針

**短期 (デプロイ前):** QRスキャンタブを非表示または削除し、「コード入力」のみを提供する。「スキャン」という選択肢自体をなくすことで、ユーザーが試みる機会を排除する。

```tsx
// QRスキャンタブを削除 → コード入力のみの1ページに変更
// もしくはタブを残す場合は "カメラを起動" ボタンを非表示にして
// 「スキャン機能は準備中」を最初から見えるラベルで表示
```

**中長期:** `html5-qrcode` または `@zxing/browser` ライブラリでカメラスキャンを実装（ISSUE-016 関連の PWA 機能と合わせて対応）。

## タスク

- [ ] コード入力のみの画面に整理
- [ ] 動作確認: `/scan` でコード入力が正常に動作すること
- [ ] 回帰確認: コード入力→参加フローが壊れていないこと

## 受け入れ条件

- 「実装中」「カメラを起動」のメッセージがユーザーに見えない
- コード入力でルームに参加できる
- スキャン機能の不在がユーザーに分かりにくい状態で公開されない

Critical
