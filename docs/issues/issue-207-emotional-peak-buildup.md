# ISSUE-207: 感情ピーク強化 — ルーレット停止前の緊張感ビルドアップ演出

📋 未着手

## カテゴリ
UX / Emotion / Animation

## 対象スコア
感情: +5 / Kano魅力品質: +1 / HEART-Happiness: +1

---

## 背景

「誰も笑わなかった夜」問題が依然として根本解決されていない。

OgoRoulette のルーレットは技術的には正確に動作する。しかし「止まる瞬間の感情ピーク」が弱い。
現在の演出フロー:

```
スピン開始 → くるくる回る → スロー → 止まる → PrismBurst + 紙吹雪 → WinnerCard表示
```

友人グループでテストした際、ルーレットが止まっても「あ、止まったね」という反応で終わった。
笑いが起きなかった。驚きもなかった。「誰が当たるかわからない」という緊張感が演出されていない。

---

## 問題

### ① 停止前の「緊張感ビルドアップ」がない

ルーレットがスローダウンする区間（残り1〜2秒）に何も起きない。
カジノのルーレットはホイールが止まる直前にテーブルが静まり返る。
スロットマシンは「リーチ」演出がある。
OgoRoulette にはその「息を呑む瞬間」がない。

### ② サウンドが停止を際立てていない

sound toggle は実装済み（ISSUE-195）だが、「ドラムロール → ジャーン（停止音）」の構成がない。
現状は回転音のみ。停止の瞬間に音楽的なクライマックスがない。

### ③ マルチプレイヤーでの「共同緊張感」がない

他のメンバーはルーレットが回っているのを自分のスマホで見ているだけ。
一体感演出（「固唾を呑む」共有体験）がない。
例：スロー区間に全員のスマホが振動する、全員に「ドキドキ」演出が出るなど。

---

## 原因

`useSpin` の `handleSlowingDown` コールバックが存在するが（play/page.tsx で受け渡し済み）、
このタイミングで追加の感情演出が実装されていない。
`handleNearMiss` も存在するがUI演出への接続がない。

---

## ユーザー影響

- リピート率低下（「また使いたい」感が弱い）
- SNSシェア率低下（「見せたい」動画が生まれない）
- バズ阻害（動画がバズるには「リアクション映え」が必要）

---

## 改善内容

### Step 1: スロービルドアップ演出（`handleSlowingDown` 時）

```tsx
// RoomPlayBody or RoomPlayOverlays でスロー開始時
const [isSlowingDown, setIsSlowingDown] = useState(false)

// handleSlowingDown コールバック内
setIsSlowingDown(true)
// → 全画面に薄い赤〜オレンジのパルスオーバーレイ表示
// → "🎯 誰が当たる...?" テキストをフェードイン
// → バイブレーション（長め）: navigator.vibrate([200, 100, 200])
```

### Step 2: 停止瞬間のスパーク演出（`handleSpinComplete` 時）

```tsx
// 現在: PrismBurst + confetti
// 追加: 停止と同時に 0.3秒の画面フラッシュ（白 → 透明）
// 追加: 停止音（ドラムロールエンド相当の短音）
// 追加: WinnerCard 表示前に 0.5秒の「ブラックアウト → reveal」
```

### Step 3: 当選者名の段階的リビール

```tsx
// 現在: WinnerCard がすぐ表示
// 変更: まず「🎉」と「当選者発表！」テキストだけ表示 → 0.8秒後に名前をスケールアップで登場
```

### Step 4: マルチプレイヤー振動（room members全員）

`room/[code]/play/page.tsx` のメンバー全員に対して、ルーレットスロー開始時に
Supabase Realtime で `event: "slowing_down"` を broadcast。
受信側でバイブレーション実行。
