# ISSUE-058: ルーレットホイールが画面内で小さく、回転の迫力が足りない

## ステータス
🟡 部分実装済み — Phase 2（グロー演出・Confetti・PrismBurst）は完了。残件: play ページのホイール max 280px 制限（home は 360px）の拡大のみ

## 優先度
**Critical**

## カテゴリ
Design / UX / Core Feature

## 概要
プレイページ（`/room/[code]/play`）のルーレットホイールが、スマートフォン画面の高さに対して相対的に小さい。ルーレットはアプリの核心機能であり「誰が奢るか決まる瞬間」の視覚的インパクトが、UX の品質を大きく左右する。現状はホイールよりも上部の情報群（ヘッダー・有効期限バナー・メンバーリスト・金額設定）が画面を支配しており、メインコンテンツが脇役になっている。

## 問題の詳細

### 画面占有率
プレイページの縦方向の分配（推定）：
```
ヘッダー            約 56px
有効期限バナー       約 40px
メンバーリスト       約 80px  （2名の場合）
金額・設定行         約 48px
─────────────────────────
上部UI 合計          約 224px
ルーレットホイール    残り領域（約 300px 程度）
SPIN ボタン          約 56px
```

iPhone 14 (844px) 想定で、ホイールが占める割合は約 35%。「ルーレットアプリ」としては存在感が弱い。

### スピン中のフィードバック
- スピン中の状態はプレイページと視覚的にほぼ同一（ホイールが回っているだけ）
- 効果音・振動・画面演出（パルス、シャドウ拡大等）がない
- 当選発表後も同じホイール表示が続き、「誰が当たった！」という瞬間の演出がない

## 改善案

### Phase 1: レイアウト調整（工数小）
1. 上部情報を折りたたみ可能にする（スピン開始後は自動折りたたみ）
2. ホイールのサイズを現状比 +20〜30% 拡大（max-width 調整）
3. SPIN ボタンをホイール内部またはホイール直下に配置し、距離を縮める

### Phase 2: 演出強化（工数中）
1. スピン中: ホイール周囲にグロー（発光）エフェクト
2. 当選直後: confetti アニメーション（`canvas-confetti` 等）
3. 当選セグメントへのポインター（矢印）を目立たせる

### Phase 3: 物理演出（工数大）
1. スピン開始時の「タメ」（少し逆回転してから加速）
2. 当選直前のスロー演出（イージング調整）

## 影響範囲
- `app/room/[code]/play/page.tsx` — レイアウト・サイズ
- `components/roulette-wheel.tsx`（または相当コンポーネント）— アニメーション
- Phase 2 以降: 外部ライブラリ追加の可能性

## 優先理由
ルーレットの視覚的インパクトはアプリの核心価値。「このアプリ楽しい！」という口コミにつながる唯一の瞬間がスピン〜当選発表であり、ここの品質が競合優位性を決める。
