# ルーム参加導線の明確化（初見ユーザーのオンボーディング改善）

## 概要

OgoRoulette のルーム機能（全員で参加して一緒に回す体験）は最大の差別化要素だが、初見ユーザーには「オーナーと参加者」「QR コードで参加する」という流れが直感的に分からない。ルーム機能に辿り着く前に離脱するユーザーが多いと推定される。

## 背景

OgoRoulette のコア体験は「みんなの画面で同時にルーレットが動く」ことにある。しかしこれを体験するには:

1. 誰かがルームを作成する（オーナー）
2. QR コードを他の参加者に見せる
3. 各参加者がスマホでスキャンして参加する
4. 全員が揃ってからオーナーがスピンする

という手順が必要。ウェルカムページには「友達みんなで参加できる」という説明はあるが、**具体的な操作の流れが見えない**。特に:
- 「QR コードをスキャンする」という操作を知らないユーザーが多い
- 「参加者全員がスマホを持つ必要がある」ことが伝わっていない
- 「オーナーだけがスピンできる」ことが分からない

## 現状の問題

1. ウェルカムページの「みんなで参加できる」セクションが静的テキスト + アイコンのみ
2. ルーム作成後の「次のステップ」が不明瞭（QR コードを見せるよう促すテキストはあるが弱い）
3. 参加者（メンバー）として参加したとき「オーナーを待っている」画面に何もない
4. 「ゲストとして参加した場合にどうなるか」が分からず、ログイン必要か不安になる

## 目的

- 初見ユーザーがルーム機能を理解し、グループで体験するまでの離脱率を下げる
- 「QR でみんなを集めてスピンする」という価値提案を直感的に伝える
- ルーム作成後の操作の迷いを減らす

## 対応内容

### 1. ウェルカムページ — ルーム機能説明の改善

現在の静的説明 → ステップ形式のミニガイドに変更:

```
Step 1: ルームを作成 → コード発行
Step 2: QR を見せる → みんなスキャン
Step 3: 全員の画面で同時にルーレット！
```

各ステップにアイコン（QrCode, Users, Sparkles）を配置。
アニメーション: 各ステップが順番にフェードインする（0.3s stagger）。

### 2. ルーム作成直後の「次にやること」ガイド

ルームを作成して `/room/[code]` に遷移した直後に:
- 「QR コードをみんなに見せてください」という大きなバナーを表示
- 参加者が 0 人の間は QR コードを前面に表示する
- 参加者が増えるたびに「○人が参加しました！」というアニメーション通知

### 3. 待機画面（メンバーがオーナーを待つ状態）の改善

現在は「準備中」の静的表示。以下を追加:
- 参加者アイコンが揺れる / 呼吸するアニメーション
- 「もうすぐスピンが始まります」というコピー（緊張感の演出）
- 他の参加者名がリアルタイムで増えていく可視化

関連: issue-134-preparing-member-loading-state.md（既存 issue と統合検討）

### 4. ゲスト参加の安心感

参加画面に「ログイン不要で参加できます」を明示。
「あなたは○○さんとして参加しています」を大きく表示。

### 5. How-to-use ページとの連携

`/how-to-use` に「ルームで使う場合」のステップを追加（GIF or 静止画）。

## 完了条件

- [ ] ウェルカムページにステップ形式のルーム機能説明が追加されている
- [ ] ルーム作成直後に「QR を見せて」という案内が大きく表示される
- [ ] 参加者が 0 人の間は QR コードが前面に出る
- [ ] メンバー待機画面に動きのある演出が加わっている
- [ ] ゲスト参加時に「ログイン不要」が明示されている

## 優先度

**Recommended** — ルーム機能への誘導が弱いと、OgoRoulette 最大の差別化体験に辿り着けないユーザーが多数発生する。

## 期待効果

- UX スコア: 65 → 69（+4）
- Growth スコア: 48 → 50（+2）
- ルーム機能の利用率向上（コア体験への到達率改善）
- グループ利用のクチコミが起きやすくなる
