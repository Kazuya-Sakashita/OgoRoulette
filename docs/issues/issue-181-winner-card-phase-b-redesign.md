# WinnerCard Phase B リデザイン（情報過多解消・CTA2択化）

## 概要

ルーレット停止後の Phase A（シネマティック演出）から Phase B（結果後アクション画面）への遷移で、画面情報量が急増しユーザーが「何をすべきか」迷う問題を解消する。Phase B のアクションを「シェア」「グループ保存」の2択に絞り込み、迷いゼロの CTA設計を実現する。

## 背景

HEART Happiness スコア（14/20）の最大の減点要因が Phase B の品質落差。Phase A（3.2秒のカウントダウン・ニアミス・停止演出）は突出したクオリティだが、Phase B に移行した瞬間にグループランキング・金額計算・シェアボタン・グループ保存・もう一度ボタン等が一斉表示されて没入が壊れる。「最高の瞬間の直後にチラシを渡される」体験になっている。

## 問題

- Phase B に表示される要素が多すぎる（グループランキング・奢り回数・金額・シェア・保存・再スピン）
- プライマリCTAが不明確（どのボタンを押せばいいか一目でわからない）
- 「グループを保存する」ボタンが Phase B の下部に埋もれており発見されにくい
- ランキング表示が情報として重いが視覚的にはコンパクトに見えない

## 目的

- Phase A → Phase B の体験落差をなくし、Happiness スコアを改善する
- 「シェア」と「グループ保存」を確実に発見・実行してもらい、Referral と Retention を改善する
- Phase B を「次の行動への自然な導線」として機能させる

## 対応内容

### Step 1: Phase B の情報設計整理

**Before（現状）:**
- 当選者名 + 奢り回数バッジ
- グループランキング（全員分）
- 金額計算結果
- シェアボタン（複数）
- グループ保存ボタン
- もう一度スピンボタン

**After（目標）:**
- 当選者名 + 奢り確定コピー（大きく1行）
- プライマリCTA: 「シェアする」（グラデーションボタン、全幅）
- セカンダリCTA: 「グループを保存」（アウトラインボタン、全幅）
- テキストリンク: 「もう一度回す」（小さく下部）
- 折りたたみ: ランキング・金額詳細（「詳細を見る ▼」で展開）

### Step 2: アニメーション設計

- Phase B 要素は stagger 0.1s で順番にフェードイン（情報が一斉に現れる衝撃を緩和）
- プライマリCTA（シェアボタン）は最後に現れ、最も目立つスケールアップ演出

### Step 3: グループ保存の強調

グループ保存ボタンを Phase B のセカンダリ CTAに格上げ。「このメンバーでまた使う → 次回1タップ」という価値を1行で説明するコピーを追加。

### Step 4: ランキング・金額の折りたたみ

デフォルト非表示→「詳細を見る」でアコーディオン展開。好奇心のあるユーザーは見られるが、シェアへの導線を邪魔しない。
